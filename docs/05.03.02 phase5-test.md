> **목적**: Sprint 2에서 완성된 Reports 페이지의 모든 구성 요소(KPI 카드, 탭, 차트, 내보내기)를 체계적으로 검증한다.
> **환경**: `npm run dev` 실행 후 `http://localhost:5173` 접속
> **전제 조건**: 보유 종목 1개 이상 등록 / 매매 일지 2건 이상 입력 완료
> **예상 소요시간**: 약 50~60분
> **작성일**: 2026-04-15
> **대상 브랜치**: `develop`
> **대상 페이지**: `/reports` (좌측 사이드바 → **리포트** 클릭)

## SCENARIO 2 — KPI 카드 4종

> **목적**: 총 수익률 / KOSPI 대비 / 거래 횟수 / 승률 카드의 데이터 정확성과 시각 상태를 검증

### 2-1. 총 수익률 카드

**테스트 단계**

1. 페이지 상단 KPI 카드 첫 번째 카드 확인.
2. 수익률 양수/음수 상태를 확인한다.

**기대 결과**

- [X] 아이콘(TrendingUp), 라벨 "총 수익률", 수익률 값(%)이 표시됨
- [X] 수익률이 **양수**이면 녹색(`text-emerald-600`) 폰트로 표시됨
- [X] 수익률이 **음수**이면 빨간색(`text-red-500`) 폰트로 표시됨
- [X] 부제목에 "손익 X원" 또는 "손익 $X" 형식의 절대 금액이 표시됨
- [X] 보유 종목이 없을 때 수익률 `0.00%` 또는 `-`로 표시됨 (오류 없음)

### 2-2. KOSPI 대비 카드

**테스트 단계**

1. 두 번째 KPI 카드를 확인한다.
2. 네트워크 연결이 정상일 때와 차단 시의 동작을 확인한다.

**기대 결과**

- [X] 라벨 "KOSPI 대비"와 초과 수익률 값(`+X.XX%` 또는 `-X.XX%`)이 표시됨
- [X] 부제목 "벤치마크 초과 수익률"이 표시됨
- [X] 초과 수익률이 양수면 녹색, 음수면 빨간색 표시
- [X] 벤치마크 데이터 로딩 중에는 값이 `-`로 표시됨
- [X] 벤치마크 API 실패 시 오류 없이 `-`로 유지됨

### 2-3. 거래 횟수 카드

**테스트 단계**

1. 세 번째 KPI 카드를 확인한다.
2. 기간 버튼을 변경하며 거래 횟수 변화를 확인한다.

**기대 결과**

- [X] 라벨 "거래 횟수"와 값(N건) 표시됨
- [X] 부제목에 "매수 X / 매도 Y" 형식으로 분류 표시됨
- [X] **오늘** 기간 선택 시 오늘 날짜 일지만 카운트됨
- [X] 일지가 없는 기간 선택 시 `0건` 표시 (오류 없음)

### 2-4. 승률 카드

**테스트 단계**

1. 네 번째 KPI 카드를 확인한다.
2. pnl(손익)이 기록된 일지가 있는 경우와 없는 경우를 비교한다.

**기대 결과**

- [X] 라벨 "승률"과 값(X%) 표시됨
- [X] pnl이 있는 일지가 있을 때: `승률 %` 값이 표시됨
- [X] pnl이 양수 건수 ÷ 전체 pnl 기록 건수 × 100 으로 계산됨
- [X] 승률 50% 이상이면 녹색 + 부제목 "양호"
- [X] 승률 50% 미만이면 빨간색 + 부제목 "개선 필요"
- [X] pnl 기록이 없으면 `-` + 부제목 "손익 기록 없음" 표시됨
