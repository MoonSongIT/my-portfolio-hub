# Phase 5 Sprint 4 — IndexedDB 최적화 & 데이터 관리 테스트 시나리오

> **목적**: Sprint 4에서 구현된 설정 페이지, 데이터 백업/복원, IndexedDB 자동 정리, 저장소 모니터링 기능을 체계적으로 검증한다.
> **환경**: `npm run dev` 실행 후 `http://localhost:5173` 접속
> **전제 조건**: 로그인 완료 상태 / 거래 내역이 1건 이상 존재 (내보내기 데이터 검증용)
> **예상 소요시간**: 약 30~40분
> **작성일**: 2026-04-17
> **대상 브랜치**: `develop`
> **대상 페이지**: `/settings` (좌측 사이드바 하단 → **설정** 클릭)

---

## 테스트 데이터 (사전 준비)

### 최소 요구 데이터

| 항목      | 조건     | 비고                             |
| --------- | -------- | -------------------------------- |
| 거래 내역 | 1건 이상 | 매매 일지에서 매수 기록 1건 추가 |
| 계좌      | 1개 이상 | 앱 최초 실행 시 자동 생성됨      |
| 관심종목  | 1개 이상 | Watchlist에서 추가               |

> ℹ️ 데이터가 부족하면 내보내기 파일에 빈 배열이 저장되어 가져오기 검증이 어렵다.
> 사전에 매매 일지 1건 이상 입력 후 테스트를 진행한다.

## SCENARIO 9 — 내보내기 → 가져오기 라운드트립

> **목적**: 내보내기 후 가져오기 복원이 데이터 손실 없이 완전히 동작하는지 통합 검증

### 9-1. 전체 라운드트립 테스트

**테스트 단계**

1. 현재 데이터 상태를 기록한다 (거래 건수, 관심종목 수).
2. **내보내기** 버튼을 클릭하여 백업 파일을 다운로드한다.
3. 브라우저 Application 탭에서 IndexedDB의 `transactions` 테이블을 직접 모두 삭제한다. (데이터 초기화 시뮬레이션)
4. **가져오기** 버튼으로 방금 다운로드한 백업 파일을 선택한다.
5. 확인 다이얼로그에서 **가져오기**를 클릭한다.
6. 매매 일지 페이지로 이동하여 데이터가 복원되었는지 확인한다.

**기대 결과**

- [X] 1단계에서 기록한 거래 건수와 복원 후 건수가 일치함
- [X] 복원된 거래 내역의 날짜, 종목, 금액 등 세부 데이터가 정확함
- [X] 관심종목이 정상 복원됨
- [X] 가져오기 toast에 표시된 백업일이 2단계의 내보내기 시각과 일치함
