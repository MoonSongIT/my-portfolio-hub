## SCENARIO 1 — SMA 이동평균선 표시 (Sprint 1.1)

> **목적**: MA20 / MA60 / MA120 라인이 캔들 위에 정상 렌더링되는지 검증

### 1-1. 기본 SMA 렌더링

**전제 조건**: 1D 타임프레임, 지표 미선택(기본) 상태

**테스트 단계**

1. 차트 영역을 확인한다.
2. 캔들스틱 위에 3개의 이동평균선이 그려져 있는지 확인한다.

**기대 결과**

- [X] MA20 : 주황색(#FF9800) 실선이 캔들 위에 표시됨
- [X] MA60 : 파란색(#2196F3) 실선이 표시됨
- [X] MA120: 보라색(#9C27B0) 실선이 표시됨
- [X] 차트 하단 범례에 `MA20 / MA60 / MA120` 색상 바 표시됨
- [X] 데이터가 20일 미만이면 MA20 라인 표시 없음 (오류 없이 정상 처리)

### 1-2. SMA 값 적정성 확인

> **목적**: 계산된 MA 값이 수학적으로 정확하고 논리적으로 타당한지 검증

**사전 준비**

1. 개발자 도구(F12) 열기 → Console 탭으로 이동
2. 다음 명령어로 차트 데이터 확인 가능 준비:
   ```javascript
   // (구현 후) 차트 데이터를 window에 노출하면 아래와 같이 확인 가능
   console.log(window.chartData)  // 또는 React DevTools에서 상태 확인
   ```

**테스트 단계**

#### Step 1: 최근 데이터 포인트에서 MA20 검증

1. 차트 우측 끝 (가장 최근 캔들)에서 마우스를 호버한다.
2. 표시되는 도구팁(Tooltip)에서 다음 정보 확인:
   - 종가(Close)
   - MA20 값
   - 호버한 캔들 인덱스

**체크항목**

- [X] 도구팁에 MA20 값이 **숫자**로 표시됨 (문자 또는 NaN 아님)
- [X] MA20 값이 **합리적인 범위**: 종가 대비 ±5% 이내 (극단적 이상치 없음)
- [X] 데이터 포인트가 20개 이상일 때만 MA20이 표시됨 (20개 미만: 표시 안 함)

#### Step 2: MA20 / MA60 / MA120 상대 위치 검증

1. 차트 중간 구간 (예: 지난 3개월 중 중간 지점)으로 마우스 이동
2. 그 시점의 세 이동평균선의 위치를 확인

**체크항목**

- [X] 세 라인이 모두 **시간순으로 왼쪽에서 오른쪽으로 일관되게 그려짐**
- [X] 어느 시점에서도 MA20, MA60, MA120이 **뒤바뀌지 않음** (계산 순서 관계없이 값 기반)
- [X] 세 라인이 주기적으로 **교차**(크로스오버) 하는 패턴이 보임
  - 예: MA20이 MA60 위에서 아래로 내려가는 구간 있음
  - 예: 추세 변화에 따라 순서가 자연스럽게 변함

#### Step 3: 이동평균선 계산 논리 검증

1. 특정 캔들(예: 가장 최근, 또는 데이터가 정확히 20/60/120개일 때)을 선택
2. 브라우저 콘솔에서 다음과 같이 해당 구간의 종가 직접 계산:
   ```javascript
   // 최근 20개 캔들 종가의 평균 = MA20 값이어야 함
   const recentCloses = chartData.slice(-20).map(d => d.close);
   const manualMA20 = recentCloses.reduce((a,b) => a+b) / recentCloses.length;
   console.log('Calculated MA20:', manualMA20);
   console.log('Chart MA20:', chartData[chartData.length-1].ma20);
   ```

**체크항목**

- [X] **계산된 값과 차트의 MA20이 거의 일치** (소수점 2~3자리까지)
  - 허용 오차: ±0.1% 이내
  - 부동소수점 연산 오차 감안
- [X] 20개 미만 데이터: MA20 필드가 `null` 또는 undefined
- [X] 60개 미만 데이터: MA60 필드가 `null` 또는 undefined
- [X] 120개 미만 데이터: MA120 필드가 `null` 또는 undefined

#### Step 4: 크로스오버 포인트 확인

1. 세 선이 교차하는 지점 (약 3~5개) 찾기
2. 각 교차 지점 전후로 값의 대소 관계 확인

**체크항목**

- [X] 교차 전: MA A > MA B
- [X] 교차 후: MA A < MA B (또는 반대)
- [X] **교차 지점이 자연스럽게 부드러운 곡선 형태** (급격한 점프나 버그 없음)

**기대 결과 요약**

- [X] MA20/60/120 값이 모두 **수치적으로 정확함** (수학 공식 검증 완료)
- [X] 세 선의 **상대 위치가 데이터 기반으로 자연스러움** (정렬 오류 없음)
- [X] **크로스오버가 자연스럽게 발생** (강제 순서 없음)
- [X] **데이터 부족 시 우아하게 처리** (오류 없이 표시 안 함)

## SCENARIO 2 — 볼린저 밴드 오버레이 (Sprint 1.2)

> **목적**: BB 토글 활성화 시 상·중·하 밴드가 캔들 위에 정상 오버레이되는지 검증

### 2-1. BB 토글 ON

**전제 조건**: 지표 컨트롤 영역에서 BB 체크박스가 꺼진(OFF) 상태

**테스트 단계**

1. 차트 우측 상단의 지표 컨트롤에서 **BB** 체크박스를 클릭한다.
2. 차트가 즉시 갱신되는지 확인한다.

**기대 결과**

- [X] BB 체크박스가 체크 상태로 변경됨
- [X] 차트에 파란 계열 반투명 3개 라인(상단·중앙·하단 밴드)이 나타남
- [X] 중앙 밴드(SMA20)가 MA20 라인과 동일한 경로로 그려짐
- [X] 상단/하단 밴드가 중앙 밴드를 대칭으로 감싸는 형태
- [X] 범례에 `BB(20,2)` 항목이 추가됨

### 2-2. BB 토글 OFF

**테스트 단계**

1. BB 체크박스를 다시 클릭하여 OFF한다.

**기대 결과**

- [X] BB 3개 라인이 모두 즉시 사라짐
- [X] 범례에서 `BB(20,2)` 항목도 사라짐
- [X] MA20/MA60/MA120 라인은 그대로 유지됨

### 2-3. BB 엣지 케이스

**테스트 단계**

1. 타임프레임을 **1M**으로 변경한 뒤 BB를 ON으로 켠다.

**기대 결과**

- [X] 월봉 기준 리샘플된 데이터에도 BB가 정상 그려짐
- [X] 기간이 짧아 데이터가 20개 미만이면 BB 라인 미표시 (오류 없음)

## SCENARIO 5 — 타임프레임 선택 (Sprint 1.5 + 1.5b)

**전제 조건**: 1D 타임프레임 상태에서 일봉 데이터가 표시 중

**테스트 단계**

1. 차트 상단 타임프레임 버튼에서 **1W**를 클릭한다.

**기대 결과**

- [X] **1W** 버튼이 파란 배경(활성 상태)으로 변경됨
- [X] 캔들 개수가 1D 대비 약 1/5로 줄어듦 (주봉으로 통합)
- [X] 각 캔들의 시간이 월요일 날짜로 표시됨 (ISO 주 기준)
- [X] 거래량 히스토그램 막대도 주봉 단위로 통합되어 표시됨 (캔들과 개수 일치)
- [ ] MA20 / MA60 / MA120 라인이 주봉 데이터 기준으로 재계산되어 표시됨

### 5-2. 1D → 1M 전환

**테스트 단계**

1. 타임프레임 버튼에서 **1M**을 클릭한다.

**기대 결과**

- [X] **1M** 버튼이 활성 상태로 변경됨
- [X] 캔들 개수가 1D 대비 약 1/20 이상으로 줄어듦 (월봉으로 통합)
- [X] 각 캔들의 시간이 해당 월의 1일로 표시됨 (예: 2026-03-01)
- [X] 거래량 히스토그램이 월 전체 거래량 합계로 표시됨 (캔들과 개수 일치)

### 5-3. 리샘플링 OHLCV 정합성 확인

**테스트 단계**

1. 1W 타임프레임에서 특정 주 캔들을 확인한다.
2. 1D 타임프레임으로 돌아가 같은 주의 일봉 5개를 확인한다.

**기대 결과**

- [X] 주봉 캔들의 **Open** = 해당 주 월요일(첫 거래일)의 일봉 Open
- [X] 주봉 캔들의 **Close** = 해당 주 금요일(마지막 거래일)의 일봉 Close
- [X] 주봉 캔들의 **High** = 해당 주 일봉 High 중 최댓값
- [X] 주봉 캔들의 **Low** = 해당 주 일봉 Low 중 최솟값
- [X] 주봉 캔들의 **거래량** = 해당 주 일봉 Volume 합산

### 5-4. 타임프레임 전환 중 지표 유지

**전제 조건**: BB, RSI, MACD 모두 ON 상태

**테스트 단계**

1. 1D → 1W → 1M → 1D 순으로 타임프레임을 전환한다.

**기대 결과**

- [X] 타임프레임 전환마다 BB, RSI, MACD 지표가 리샘플링된 데이터 기준으로 재계산됨
- [X] 지표 ON/OFF 상태가 타임프레임 전환 후에도 유지됨
- [X] 오류 없이 매 전환마다 즉시 갱신

### 5-5. 타임프레임 전환 시 거래량-캔들 정렬 확인

2. 1W로 전환 후 다시 확인한다.

## SCENARIO 6 — 매매 일지 마커 (Sprint 1.6)

> **목적**: 매매 일지에 기록된 매수/매도 이력이 차트 위에 마커로 정상 표시되는지 검증

### 6-1. 매수 마커 표시 확인

**전제 조건**: 삼성전자 종목 상세 페이지, 매매 일지에 매수 기록 1개 이상 존재

**테스트 단계**

1. 캔들스틱 차트에서 매수 기록이 있는 날짜를 찾는다.
2. 해당 날짜의 캔들 하단을 확인한다.

**기대 결과**

- [X] 초록색 위쪽 화살표(▲) 마커가 캔들 하단(belowBar)에 표시됨
- [X] 마커에 `매수 N주 | 심리유형` 텍스트가 표시됨 (예: `매수 10주 | 분할매수 원칙`)
- [X] 심리유형이 없는 경우 `매수 N주`만 표시됨

### 6-2. 매도 마커 표시 확인

**전제 조건**: 매매 일지에 매도 기록 1개 이상 존재

**테스트 단계**

1. 캔들스틱 차트에서 매도 기록이 있는 날짜의 캔들 상단을 확인한다.

**기대 결과**

- [X] 빨간색 아래쪽 화살표(▼) 마커가 캔들 상단(aboveBar)에 표시됨
- [X] 마커에 `매도 N주 | 심리유형` 텍스트 표시됨

### 6-3. 동일 날짜 복수 거래 병합

**전제 조건**: 같은 날짜에 매수 거래 2건 이상이 존재

**테스트 단계**

1. 동일 날짜에 매수 2건을 입력한다 (예: 수량 5주 + 3주).
2. 해당 날짜의 차트 마커를 확인한다.

**기대 결과**

- [X] 마커가 2개가 아닌 1개로 병합되어 표시됨
- [X] 텍스트에 합산 수량이 표시됨 (예: `매수 8주`)

### 6-4. 같은 날 매수 + 매도 동시 존재

**전제 조건**: 같은 날짜에 매수 1건과 매도 1건 모두 존재

**테스트 단계**

1. 동일 날짜에 매수와 매도를 각 1건씩 입력한다.
2. 해당 날짜의 차트 마커를 확인한다.

**기대 결과**

- [X] 캔들 하단에 초록 매수 마커 1개 표시됨
- [X] 캔들 상단에 빨간 매도 마커 1개 표시됨
- [X] 두 마커가 겹치지 않고 각각 위/아래에 표시됨

### 6-5. 다른 종목 마커 미표시

**테스트 단계**

1. Apple(AAPL) 종목 상세 페이지로 이동한다.

**기대 결과**

- [X] Apple에 매매 기록이 없으면 마커 없음

### 6-6. 매매 일지 추가 후 마커 자동 갱신

**테스트 단계**

1. 삼성전자 차트가 열린 상태에서, **별도 탭 없이** 매매 일지 입력 모달을 열어 새 매수 기록을 추가한다.

**기대 결과**

### 6-7. 주봉 타임프레임에서 마커 표시

- [ ] 마커가 캔들이 없는 곳에 떠 있거나 오류가 발생하지 않음

## SCENARIO 7 — 기간 선택 버튼 (Sprint 1.7)

> **목적**: 1M/3M/6M/1Y/ALL 기간 버튼 클릭 시 차트 visible range가 올바르게 변경되는지 검증

### 7-1. 기간 버튼 UI 확인

**테스트 단계**

1. 차트 상단 컨트롤 바를 확인한다.

**기대 결과**

- [ ] `[1D | 1W | 1M]` 타임프레임 버튼 오른쪽에 구분선(|)이 있음
- [ ] 구분선 오른쪽에 `[1M | 3M | 6M | 1Y | ALL]` 기간 버튼이 표시됨
- [ ] 기본 선택: `ALL` 버튼이 파란 배경(활성)으로 표시됨

### 7-2. ALL → 1Y 전환

**테스트 단계**

1. 기간 버튼에서 **1Y**를 클릭한다.

**기대 결과**

- [ ] **1Y** 버튼이 활성 상태로 변경됨
- [ ] 차트 x축이 최근 365일 범위로 줄어듦
- [ ] 캔들 데이터가 삭제되지 않고 화면 범위만 변경됨 (스크롤하면 이전 데이터 접근 가능)
- [ ] 차트 재생성(깜박임) 없이 부드럽게 전환됨

### 7-3. 1Y → 6M → 3M → 1M 순차 전환

**테스트 단계**

1. 1Y → 6M → 3M → 1M 순으로 클릭한다.

**기대 결과**

- [ ] 각 전환마다 활성 버튼이 바뀜
- [ ] 1M 선택 시 최근 30일 범위만 표시됨 (캔들 개수가 약 20~22개)
- [ ] 매 전환마다 차트가 재생성 없이 즉시 범위 변경됨

### 7-4. 1M → ALL 복귀

**테스트 단계**

1. **ALL** 버튼을 클릭한다.

**기대 결과**

- [ ] `fitContent()` 호출로 전체 데이터가 화면에 맞춰 표시됨
- [ ] 모든 캔들이 한 화면에 보임
- [ ] **ALL** 버튼이 활성 상태

### 7-5. 기간 선택 + 타임프레임 변경 조합

**테스트 단계**

1. 기간을 **3M**으로 선택한다.
2. 타임프레임을 **1D** → **1W**로 변경한다.

**기대 결과**

- [ ] 타임프레임 변경 후에도 기간 3M 버튼이 활성 상태를 유지함
- [ ] 차트 재생성 후 `applyPeriod()`가 다시 호출되어 3M 범위로 표시됨
- [ ] 오류 없이 정상 전환

### 7-6. 기간 선택 + 지표 토글 조합

**테스트 단계**

1. 기간 **6M** 선택 → RSI 토글 ON → MACD 토글 ON

**기대 결과**

- [ ] 지표 ON/OFF마다 차트가 재생성되지만 6M 기간 설정이 복원됨
- [ ] RSI, MACD 패널이 추가되어도 기간 표시가 유지됨
