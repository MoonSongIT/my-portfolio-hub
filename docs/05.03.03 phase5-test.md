# Phase 5 Sprint 3 — Watchlist 페이지 완성 테스트 시나리오

> **목적**: Sprint 3에서 완성된 Watchlist 페이지의 모든 기능(스파크라인, 알림 트리거, 정렬/필터, 메모)을 체계적으로 검증한다.
> **환경**: `npm run dev` 실행 후 `http://localhost:5173` 접속
> **전제 조건**: 로그인 완료 상태 / 관심종목 3개 이상 등록 (KRX, KOSDAQ, US 각 1개 이상 포함)
> **예상 소요시간**: 약 40~50분
> **작성일**: 2026-04-17
> **대상 브랜치**: `develop`
> **대상 페이지**: `/watchlist` (좌측 사이드바 → **관심종목** 클릭)

## SCENARIO 1 — 페이지 헤더 & 기본 레이아웃

> **목적**: 페이지 헤더, 종목 수 요약, 알림 개수, 버튼 구성이 정상 렌더링되는지 검증

### 1-1. 헤더 렌더링

**테스트 단계**

1. `/watchlist` 페이지 접속 후 상단을 확인한다.

**기대 결과**

- [X] 좌측 상단에 **"관심종목"** (H2) 타이틀이 표시됨
- [X] 부제목에 **"N종목 모니터링 중"** 형식으로 종목 수가 표시됨
- [X] 알림이 1개 이상 있으면 **"· 알림 N개 활성"** 문구가 파란색으로 표시됨
- [X] 우측에 **오늘 브리핑 / 정렬 / 새로고침 / 뷰 토글 / 추가** 버튼이 순서대로 표시됨

### 1-2. 카드/테이블 뷰 토글

**테스트 단계**

1. 우측 상단의 **격자 아이콘(카드 뷰)** 버튼을 클릭한다.
2. **리스트 아이콘(테이블 뷰)** 버튼을 클릭한다.

**기대 결과**

- [X] 카드 뷰: 격자 아이콘이 파란색 배경으로 활성화, 종목이 카드 형태(2~4열 그리드)로 표시됨
- [X] 테이블 뷰: 리스트 아이콘이 파란색 배경으로 활성화, 종목이 테이블 행으로 표시됨
- [X] 뷰 전환 시 데이터가 유지됨 (종목 수, 가격 동일)

### 1-3. 빈 상태 (관심종목 없을 때)

> **주의**: 관심종목을 모두 삭제한 후 확인하는 엣지 케이스 테스트

**테스트 단계**

1. 기존 관심종목을 모두 삭제한다.
2. 빈 상태 화면을 확인한다.
3. **"첫 종목 추가하기"** 버튼을 클릭한다.
4. 삭제한 종목을 다시 추가하여 원래 상태로 복구한다.

**기대 결과**

- [X] Bell 아이콘 + **"관심종목이 없습니다"** 텍스트가 중앙에 표시됨
- [X] 안내 문구: "종목을 추가하여 가격 변동을 모니터링하세요"가 표시됨
- [X] **"첫 종목 추가하기"** 버튼 클릭 시 종목 추가 모달이 열림

## SCENARIO 2 — 종목 추가 모달

> **목적**: 검색 기반 관심종목 추가 기능이 정상 동작하는지 검증

### 2-1. 모달 열기 및 검색

**테스트 단계**

1. 우측 상단 **추가** 버튼을 클릭한다.
2. 검색 입력창에 **"삼성"** 을 입력한다.
3. 자동완성 결과를 확인한다.

**기대 결과**

- [X] 모달이 열리면 검색 입력창에 포커스가 자동으로 이동함
- [X] "삼성" 입력 후 0.3초 내 검색 결과가 드롭다운으로 표시됨
- [X] 각 결과에 종목명 / 티커 · 시장 정보가 표시됨
- [X] 이미 추가된 종목에는 **"추가됨"** 배지가 표시되고 버튼이 비활성화됨

### 2-2. 종목 추가

**테스트 단계**

1. 아직 추가되지 않은 종목 검색 결과를 클릭한다.

**기대 결과**

- [X] 클릭 즉시 모달이 닫힘
- [X] 화면 우하단에 **"[종목명] 관심종목 추가"** 초록색 toast가 표시됨
- [X] 관심종목 목록에 해당 종목이 추가됨
- [X] 헤더 부제목의 종목 수가 1 증가함

### 2-3. 중복 추가 방지

**테스트 단계**

1. 이미 추가된 종목을 다시 검색한다.

**기대 결과**

- [X] 해당 종목 결과에 **"추가됨"** 배지가 표시됨
- [X] 클릭해도 중복 추가되지 않음

## SCENARIO 3 — 미니 스파크라인 차트

> **목적**: 카드 및 테이블 뷰에 1개월 가격 추이 스파크라인이 정상 렌더링되는지 검증

### 3-1. 카드 뷰 스파크라인

**전제 조건**: 카드 뷰 모드, 관심종목 3개 이상

**테스트 단계**

1. 카드 뷰에서 각 종목 카드 우측 하단 영역을 확인한다.

**기대 결과**

- [X] 각 카드에 소형 SVG 라인 차트가 표시됨 (약 88×36px)
- [X] 등락률이 양수(+)인 종목: 스파크라인이 **빨간색** 선으로 표시됨 (한국 주식 관례)
- [X] 등락률이 음수(-)인 종목: 스파크라인이 **초록색** 선으로 표시됨 (한국 주식 관례)
- [ ] 라인 아래에 반투명 채움 영역이 표시됨
- [ ] 가장 최근 데이터 포인트에 원형 점(dot)이 표시됨
- [ ] 히스토리 데이터 로딩 전에는 회색 직사각형(placeholder)이 표시됨

### 3-2. 테이블 뷰 스파크라인

**테스트 단계**

1. 테이블 뷰로 전환한다.
2. **"추이"** 컬럼을 확인한다.

**기대 결과**

- [ ] 데스크탑 화면(md 이상)에서 **"추이"** 컬럼에 축소된 스파크라인이 표시됨 (약 72×28px)
- [ ] 색상 로직이 카드 뷰와 동일하게 적용됨
- [ ] 모바일 화면에서는 추이 컬럼이 숨겨짐 (overflow 없음)

### 3-3. 스파크라인 데이터 적정성

**테스트 단계**

1. KRX 종목(삼성전자)과 US 종목(NVDA) 스파크라인을 비교한다.

**기대 결과**

- [ ] KRX 종목: 네이버 API 기반 최근 1개월 종가로 그려짐
- [ ] US 종목: Yahoo Finance 기반 최근 1개월 종가로 그려짐
- [ ] 두 종목 모두 가격 추이가 단순 직선이 아닌 자연스러운 곡선/꺾선으로 표시됨
- [ ] 데이터가 2개 미만인 경우 스파크라인 대신 회색 placeholder가 표시됨 (오류 없음)

### 3-4. 스파크라인 캐시 동작

**테스트 단계**

1. 새로고침 버튼을 클릭한다. (batchQuotes만 재조회)
2. 스파크라인 변화를 관찰한다.

**기대 결과**

- [ ] 새로고침 버튼 클릭 시 현재가/등락률은 즉시 재조회됨
- [ ] 스파크라인은 1시간 캐시로 인해 즉시 변경되지 않음 (정상 동작)
- [ ] 페이지 새로고침(F5) 이후에는 스파크라인도 다시 fetch됨

## SCENARIO 4 — 가격 알림 트리거 Toast

> **목적**: 가격 조건이 충족되었을 때 toast 알림이 정상 발생하는지 검증

### 4-1. 알림 트리거 Toast 발생

**전제 조건**: 사전 준비 단계에서 삼성전자에 **이하(↓) / 목표가 1원** 알림이 등록되어 있어야 함
(현재가가 1원보다 훨씬 높으므로 조건이 항상 충족됨)

**테스트 단계**

1. `/watchlist` 페이지에 접속 또는 새로고침한다.
2. 가격 데이터가 로드될 때까지 기다린다 (약 3~10초).

**기대 결과**

- [ ] 화면 우하단에 **주황색(warning) toast**가 표시됨
- [ ] Toast 제목: **"📉 삼성전자 알림 도달"** 형식
- [ ] Toast 설명: **"현재가 ₩XXX,XXX — 목표가 ₩1 이하"** 형식
- [ ] Toast가 약 8초 후 자동 닫힘 (또는 닫기 버튼으로 수동 닫기)

### 4-2. 중복 알림 방지

**테스트 단계**

1. 가격 알림 toast가 발생한 직후 **새로고침** 버튼을 클릭한다.

**기대 결과**

- [ ] 새로고침(batchQuotes 재조회) 후에도 같은 알림 toast가 **재발생하지 않음**
- [ ] 페이지 새로고침(F5) 이후에는 다시 toast가 발생함 (ref가 초기화되므로 정상)

### 4-3. 알림 조건별 동작

**테스트 단계**

1. AlarmDialog에서 **이상(↑) / 목표가 999,999,999원** 알림을 추가한다. (절대 도달 불가)
2. 가격이 로드되어도 해당 알림이 발생하지 않는지 확인한다.

**기대 결과**

- [ ] 목표가 999,999,999원 이상(↑) 알림은 toast가 발생하지 않음
- [ ] 삼성전자 1원 이하(↓) 알림만 toast가 발생함
- [ ] 두 알림이 별개로 독립적으로 관리됨

### 4-4. 알림 설정 다이얼로그

**테스트 단계**

1. 카드 뷰에서 **알림 설정** 버튼을 클릭한다.
2. 조건(이상/이하)을 전환하고, 목표가를 입력한 후 **알림 추가** 버튼을 클릭한다.
3. 등록된 알림 목록을 확인하고 삭제(휴지통 아이콘)를 클릭한다.

**기대 결과**

- [ ] 다이얼로그에 현재가가 표시됨 (현재가 > 0인 경우)
- [ ] 이상(↑) 선택 시 초록색, 이하(↓) 선택 시 빨간색 버튼이 활성화됨
- [ ] 유효하지 않은 가격(0, 빈값) 입력 시 에러 메시지 표시
- [ ] 알림 추가 후 "등록된 알림" 목록에 즉시 표시됨
- [ ] 삭제 버튼 클릭 시 해당 알림만 제거됨

## SCENARIO 5 — 정렬 기능

> **목적**: 정렬 드롭다운의 5가지 옵션이 정확하게 작동하는지 검증

### 5-1. 정렬 패널 토글

**테스트 단계**

1. 헤더 우측의 **슬라이더 아이콘(SlidersHorizontal)** 버튼을 클릭한다.
2. 정렬 패널이 열리는지 확인한다.
3. 패널 외부 영역을 클릭하거나 옵션을 선택해 패널이 닫히는지 확인한다.

**기대 결과**

- [ ] 버튼 클릭 시 드롭다운 패널이 버튼 아래에 표시됨
- [ ] 패널에 5가지 정렬 옵션 표시: **등락률 ↓ / 등락률 ↑ / 이름 순 / 현재가 ↓ / 추가일 순**
- [ ] 현재 선택된 옵션이 파란색 배경으로 강조됨 (기본: 등락률 ↓)
- [ ] 옵션 선택 시 패널이 자동으로 닫힘

### 5-2. 등락률 내림차순 정렬 (기본값)

**테스트 단계**

1. 정렬 옵션에서 **"등락률 ↓"** 를 선택한다.

**기대 결과**

- [ ] 가장 많이 오른 종목이 맨 앞에 표시됨
- [ ] 등락률 높은 순으로 카드/테이블이 정렬됨
- [ ] 등락률 데이터가 없는(---) 종목은 맨 뒤에 배치됨

### 5-3. 등락률 오름차순 정렬

**테스트 단계**

1. 정렬 옵션에서 **"등락률 ↑"** 를 선택한다.

**기대 결과**

- [ ] 가장 많이 하락한 종목(음수 최대)이 맨 앞에 표시됨
- [ ] 이전 "등락률 ↓" 정렬의 역순으로 표시됨

### 5-4. 이름 순 정렬

**테스트 단계**

1. 정렬 옵션에서 **"이름 순"** 을 선택한다.

**기대 결과**

- [ ] 종목명이 한국어 사전 순(가나다 순)으로 정렬됨
- [ ] 한글 → 영문 순서로 표시됨

### 5-5. 현재가 내림차순 정렬

**테스트 단계**

1. 정렬 옵션에서 **"현재가 ↓"** 를 선택한다.

**기대 결과**

- [ ] 현재가가 가장 높은 종목이 맨 앞에 표시됨
- [ ] KRW와 USD가 혼재하는 경우 숫자 값 기준으로 정렬됨 (통화 변환 없이 원시 값 비교)
- [ ] 현재가가 0인 종목은 맨 뒤에 배치됨

### 5-6. 추가일 순 정렬

**테스트 단계**

1. 정렬 옵션에서 **"추가일 순"** 을 선택한다.

**기대 결과**

- [ ] 가장 최근에 추가한 종목이 맨 앞에 표시됨 (내림차순)
- [ ] 방금 추가한 종목이 첫 번째 위치에 표시됨

## SCENARIO 6 — 시장 필터 탭

> **목적**: 시장 필터 탭이 정확한 종목만 표시하는지 검증

### 6-1. 필터 탭 렌더링

**전제 조건**: KRX 1종목, KOSDAQ 1종목, NASDAQ 1종목 이상 등록 상태

**테스트 단계**

1. 헤더 하단의 필터 탭 영역을 확인한다.

## SCENARIO 7 — 메모 기능

> **목적**: 카드별 인라인 메모 편집이 정상 동작하는지 검증

### 7-1. 메모 추가

**테스트 단계**

1. 카드 뷰에서 임의의 종목 카드 하단의 **"메모 추가..."** (이탤릭 텍스트)를 클릭한다.
2. 텍스트 입력창이 나타나면 메모를 입력한다 (예: "목표가 85,000원").
3. **Enter** 키 또는 **✓ 버튼** 을 클릭한다.

**기대 결과**

- [ ] 클릭 즉시 입력 필드가 나타나고 포커스가 이동함
- [ ] 메모 입력 후 Enter/✓ 클릭 시 편집 모드 종료
- [ ] 카드에 입력한 메모 텍스트가 회색으로 표시됨
- [ ] 새로고침(F5) 후에도 메모가 유지됨 (LocalStorage 저장)

### 7-2. 메모 수정

**테스트 단계**

1. 이미 메모가 입력된 카드에서 메모 텍스트 위에 마우스를 올린다.
2. 연필 아이콘(Pencil)이 나타나면 텍스트를 클릭한다.
3. 기존 메모를 수정 후 Enter를 누른다.

**기대 결과**

- [ ] 마우스 호버 시 연필 아이콘이 애니메이션으로 나타남
- [ ] 클릭 시 기존 메모 내용이 입력창에 채워진 상태로 편집 모드 전환
- [ ] 수정된 메모가 즉시 반영됨

### 7-3. 메모 취소

**테스트 단계**

1. 메모 편집 모드에서 내용을 변경한 후 **Escape** 키 또는 **✕ 버튼** 을 클릭한다.

**기대 결과**

- [ ] 편집 모드 종료 후 이전 메모 내용이 복원됨 (변경 내용 미저장)

### 7-4. 메모 삭제 (빈 값 저장)

**테스트 단계**

1. 메모 편집 모드에서 내용을 모두 지우고 Enter를 누른다.

**기대 결과**

- [ ] 메모가 삭제되고 다시 **"메모 추가..."** 이탤릭 텍스트로 표시됨

### 7-5. 메모 최대 길이

**테스트 단계**

1. 메모 편집 모드에서 80자 이상 입력을 시도한다.

**기대 결과**

- [ ] 80자 초과 입력이 차단됨 (`maxLength={80}` 적용)

### 7-6. 테이블 뷰에서 메모 표시

**테스트 단계**

1. 테이블 뷰로 전환한다.
2. 메모가 입력된 종목의 행을 확인한다.

**기대 결과**

- [ ] 종목명 아래에 메모가 작은 회색 글씨로 표시됨
- [ ] 메모가 없는 종목은 종목명만 표시됨

## SCENARIO 8 — 활성 알림 요약 카드

> **목적**: 페이지 하단의 알림 요약 섹션이 정확하게 표시되는지 검증

### 8-1. 알림 요약 렌더링
