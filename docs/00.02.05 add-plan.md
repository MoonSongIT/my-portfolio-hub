## 추가 작업 개요

### 왜 추가하는가

기존 Phase 2~4는 일반적인 포트폴리오 관리 앱 기준으로 구현되었다.
프로젝트 목적이 **"매매 심리를 기록하고 AI로 돌아보는 투자 성장 도구"** 로 재정의됨에 따라

## STEP 1 — journalStore 구현

> **파일**: `src/store/journalStore.js` (신규 생성)
> **패턴**: 기존 `portfolioStore.js`와 동일하게 Zustand + persist + immer 사용

### 데이터 구조

```javascript
// 일지 항목 1건 구조
{
  id: 'uuid-...',           // 고유 ID (crypto.randomUUID())
  date: '2026-04-03',       // 거래 날짜 (YYYY-MM-DD)
  ticker: '005930',          // 종목 코드
  name: '삼성전자',           // 종목명
  market: 'KRX',            // 시장 (KRX / NASDAQ / NYSE)
  action: 'buy',            // 매수(buy) / 매도(sell)
  price: 72000,             // 거래가
  quantity: 10,             // 수량
  amount: 720000,           // 거래 금액 (price * quantity)
  psychology: '추격매매',    // 심리 카테고리 (선택형)
  memo: '...',              // 메모 (선택 사항)
  pnl: null,                // 실현 손익 (매도 시 입력, null 허용)
  createdAt: '2026-04-03T09:30:00.000Z',
}
```

### 심리 카테고리 상수

```javascript
export const BUY_PSYCHOLOGY = [
  '미래가치 투자',
  '분할매수 원칙',
  '추격매매',
  '뉴스 편승',
  '저가 매수',
  '목표가 도달',
  '기타',
]

export const SELL_PSYCHOLOGY = [
  '목표가 실현',
  '손절 원칙',
  '공포에 매도',
  '수익 실현 (조급)',
  '리밸런싱',
  '기타',
]
```

### 구현할 액션 & 셀렉터

```javascript
// 액션
addEntry(entry)                        // 일지 추가
updateEntry(id, updates)               // 일지 수정
deleteEntry(id)                        // 일지 삭제

// 셀렉터
getEntriesByTicker(ticker)             // 종목별 조회
getEntriesByPsychology(psychology)     // 심리 유형별 조회
getEntriesByDateRange(start, end)      // 기간별 조회
getProfitByPsychology()                // 심리 유형별 수익률 집계
getSummaryStats()                      // 전체 통계 요약
```

### getProfitByPsychology 반환 형식

```javascript
// 반환 예시
[
  { psychology: '미래가치 투자', count: 5, totalPnl: 1250000, avgPnlRate: 12.4 },
  { psychology: '추격매매',      count: 4, totalPnl: -320000, avgPnlRate: -5.2 },
  { psychology: '공포에 매도',   count: 2, totalPnl: -180000, avgPnlRate: -8.1 },
]
```

## STEP 2 — Journal 컴포넌트 구현

> **디렉토리**: `src/components/journal/` (신규 생성)

### 2-1. PsychologySelector.jsx

심리 카테고리를 선택하는 버튼 그룹 컴포넌트.

```
[미래가치 투자] [분할매수 원칙] [추격매매] [뉴스 편승] [저가 매수] [목표가 도달] [기타]
```

- `action` prop으로 `buy` / `sell` 구분 → 해당 카테고리 목록 렌더링
- 선택 시 버튼 색상 강조 (선택: 파란색 채움 / 미선택: 테두리만)
- `value` / `onChange` prop으로 상위 폼과 연동

### 2-2. JournalEntryForm.jsx

**빠른 입력 모드** — 매매 직후 즉시 기록용

```
┌─────────────────────────────────────────┐
│ ● 새 매매 기록                           │
│                                         │
│ 종목: [삼성전자 검색...        ]         │
│ 구분: ( 매수 ) ( 매도 )                  │
│ 날짜: [2026-04-03  ]  가격: [72,000   ] │
│ 수량: [10        ]                      │
│                                         │
│ 매수 심리                                │
│ [미래가치] [분할매수] [추격매매] [뉴스편승]│
│ [저가매수] [목표가도달] [기타]            │
│                                         │
│ 메모 (선택)                              │
│ [                              ]        │
│                                         │
│               [취소] [저장]              │
└─────────────────────────────────────────┘
```

- shadcn/ui `Dialog` 또는 인라인 폼으로 구현
- 종목 검색: 기존 `krxStocks.js` 데이터 활용 (Command 컴포넌트)
- 필수 필드: 종목, 구분, 날짜, 가격, 수량, 심리 카테고리
- 선택 필드: 메모
- 저장 시 `journalStore.addEntry()` 호출

### 2-3. JournalBatchForm.jsx

**일괄 입력 모드** — 마감 후 당일 거래 한 번에 입력

- 날짜 선택 피커 (기본값: 오늘)
- "+ 거래 추가" 버튼으로 입력 행 추가
- 한 화면에 여러 건 나열 → 한 번에 전체 저장
- 각 행: JournalEntryForm의 필드와 동일 (수평 레이아웃으로 압축)

### 2-4. JournalList.jsx

일지 목록 조회 및 관리.

│ │                              [수정][삭제]│  │

## STEP 4 — JournalCoachAgent 구현

> **파일**: `src/agents/journalCoachAgent.js` (신규 생성)
> **패턴**: 기존 `portfolioAgent.js` 참조

### 시스템 프롬프트 구조

```javascript
export function buildJournalCoachPrompt(journalContext) {
  return `당신은 사용자의 투자 매매 일지를 분석하는 투자 심리 코치입니다.

아래는 사용자의 매매 일지 데이터입니다:
${journalContext}

분석 프레임워크:
1. 심리 유형별 수익률: 어떤 심리로 매수/매도했을 때 수익이 났는가
2. 반복 실수 패턴: 같은 실수가 몇 번 반복되었는가
3. 잘된 결정 강화: 좋은 결과를 낸 심리 유형과 상황
4. 개선 제안: 구체적이고 실천 가능한 행동 변화

응답 규칙:
- 데이터에 없는 내용 추측 금지
- 직접 매수/매도 권유 금지
- 공감적 어조 유지 (비판보다 성장 관점)
- 면책 문구 필수: "이 분석은 참고용이며 투자 결정의 책임은 본인에게 있습니다."`
}
```

### 컨텍스트 빌더
