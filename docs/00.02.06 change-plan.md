## 왜 재설계하는가

현재 구조의 핵심 모순:

```
journalStore  ──── entries (accountId 없음)
portfolioStore ─── accounts → holdings (수동 입력)
```

- **journalStore**: 거래를 기록하지만 "어느 계좌"인지 모름
- **portfolioStore**: 보유 현황을 수동으로 입력 (거래내역과 무관)
- 두 store가 완전히 단절 → 같은 거래를 두 곳에 이중 입력해야 함

### 목표 구조

```
accountStore ──── 계좌 목록 (CRUD)
     ↓
journalStore ──── 거래내역 (accountId 포함)
     ↓ (자동 파생)
포트폴리오 현황 ── 거래내역에서 계산 (수동 입력 불필요)
```

## 파생 계산 로직 (핵심)

### 평균단가 계산 (가중평균법)

```
매수 100주 @ 70,000 → 잔고: 100주, 총원가: 7,000,000
매수  50주 @ 80,000 → 잔고: 150주, 총원가: 11,000,000, 평균단가: 73,333
매도  50주         → 잔고: 100주, 총원가: 7,333,300
                     실현손익: (매도가 - 73,333) × 50주
```

### computeHoldings 흐름

```
1. accountId로 entries 필터
2. 날짜 오름차순 정렬
3. ticker별 매수 누적 → 총원가, 수량 증가
4. ticker별 매도 → 평균단가 기준 총원가 감소, 수량 감소
5. 수량 > 0인 포지션만 반환
```

## STEP 2 — journalStore 수정

> **파일**: `src/store/journalStore.js` (수정)

### 변경 사항

1. **entry 스키마에 `accountId`, `fee` 필드 추가**

```javascript
addEntry: (entry) => set((state) => {
  state.entries.push({
    id: crypto.randomUUID(),
    createdAt: new Date().toISOString(),
    fee: 0,      // ← 신규 기본값
    pnl: null,
    memo: '',
    ...entry,    // accountId 포함 필수
  })
})
```

2. **computeHoldings(accountId) 셀렉터 추가**

```javascript
computeHoldings: (accountId) => {
  const entries = get().entries
    .filter(e => e.accountId === accountId)
    .sort((a, b) => a.date.localeCompare(b.date))

  const map = {}

  for (const e of entries) {
    if (!map[e.ticker]) {
      map[e.ticker] = { ticker: e.ticker, name: e.name, market: e.market, quantity: 0, totalCost: 0 }
    }
    const pos = map[e.ticker]
    if (e.action === 'buy') {
      pos.totalCost += e.price * e.quantity + (e.fee || 0)
      pos.quantity  += e.quantity
    } else {
      const avgPrice = pos.quantity > 0 ? pos.totalCost / pos.quantity : 0
      pos.pnl = (e.price - avgPrice) * e.quantity - (e.fee || 0)
      pos.totalCost -= avgPrice * e.quantity
      pos.quantity  -= e.quantity
    }
  }

  return Object.values(map)
    .filter(p => p.quantity > 0)
    .map(p => ({ ...p, avgPrice: p.quantity > 0 ? Math.round(p.totalCost / p.quantity) : 0 }))
},
```

3. **computeAllHoldings() 셀렉터 추가** (전체 계좌 통합)

4. **기존 셀렉터 accountId 필터 파라미터 추가** (선택적, 없으면 전체)

5. **migrate 버전 업** (version: 2)
   - 기존 entries에 `accountId: 'default'`, `fee: 0` 자동 부여

## 마이그레이션 전략

### 기존 데이터 처리

| 데이터 | 처리 방법 |
|--------|----------|
| journalStore.entries (accountId 없음) | migrate 시 `accountId: 'default'`, `fee: 0` 자동 부여 |
| portfolioStore.accounts (수동 holdings) | **무시** (거래내역과 무관한 데이터이므로 제거) |
| accountStore | 신규 생성, "기본 계좌" 자동 생성 |

### migrate 시나리오

1. 앱 실행 시 `accountStore`에 계좌 없으면 → `AccountSetupModal` 자동 표시
2. 기존 journalStore entries가 있는데 `accountId`가 없는 경우 → `'default'` 부여
3. `accountStore`에 id=`'default'`인 계좌 자동 생성 ("기본 계좌")
