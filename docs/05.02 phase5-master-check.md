## Sprint 1.5b — 주봉/월봉 리샘플링 (Weekly/Monthly Resampling)

> 브랜치: `feature/phase5-chart-resampling`
> 예상 기간: 1일
> 목적: TimeframeSelector에서 1W/1M 전환 시 실시간 OHLCV 데이터를 주/월 단위로 정확히 리샘플링

### 1.5b.1 주/월봉 리샘플링 로직

- ✅ `src/utils/technicalIndicators.js`에 `resampleOHLCV(candles, timeframe)` 함수 추가
  - 입력: 일봉 OHLCV 배열 `[{time, open, high, low, close, volume}, ...]`
  - 출력: 주봉/월봉 `[{time, open, high, low, close, volume}, ...]`
  - 로직:
    - `timeframe === '1W'`: ISO 주 기준 묶음 (월요일 시작), 해당 주의 Open(첫값)/Close(마지막값), High/Low(최고/최저), Volume(합산)
    - `timeframe === '1M'`: 월 기준 묶음, 해당 월의 Open(첫값)/Close(마지막값), High/Low(최고/최저), Volume(합산)
- ✅ 단위 테스트: 일봉 20일분 → 주봉 4주 리샘플링 검증 (테스트 4/4 통과)

### 1.5b.2 TimeframeSelector 연동

- ✅ `src/components/charts/CandlestickChart.jsx` 수정
  - `timeframe` dependency 추가 → useEffect 재실행 트리거
  - `resampleOHLCV(dailyData, timeframe)` 호출 (1D 제외)
  - 리샘플된 OHLCV를 `chart.addSeries()` 데이터로 적용
  - BB/RSI/MACD 지표도 리샘플된 데이터 기반으로 재계산
- ✅ **[CRITICAL 수정]** 거래량 히스토그램 미정렬 (volumeData → chartData 변경)
  - 문제: timeframe 변경 시 candleData는 리샘플링되지만 volumeData는 원본 data 사용
  - 해결: volumeData 소스를 `data` → `chartData`로 변경 (일괄 적용)
  - 변경 사항: `volumeData = chartData.filter().map()` (line 171)

### 1.5b.3 리샘플링 검증

- ✅ 브라우저 직접 확인: 1D → 1W 전환 시 촛대 개수 감소 (20 → 4개)
- ✅ 브라우저 직접 확인: 1D → 1M 전환 시 촛대 개수 감소 (20 → 1개)
- ✅ 브라우저 직접 확인: 리샘플된 데이터의 Open/Close 정상 표시
- ✅ 빌드 성공 (npm run build 0 errors)
