# Phase 5 Sprint 5 — PWA 강화 & 오프라인 테스트 시나리오

> **목적**: Sprint 5에서 구현된 오프라인 감지 배너, AI 채팅 오프라인 차단, PWA 업데이트 알림 기능을 체계적으로 검증한다.
> **환경**: `npm run dev` 실행 후 `http://localhost:5173` 접속
> **전제 조건**: 로그인 완료 상태
> **예상 소요시간**: 약 15~20분
> **작성일**: 2026-04-17
> **대상 브랜치**: `develop`

---

## 테스트 환경 준비

### 오프라인 시뮬레이션 방법

브라우저 DevTools를 이용하여 오프라인 상태를 시뮬레이션한다.

1. Chrome DevTools 열기 (F12)
2. **Network** 탭 선택
3. **Throttling** 드롭다운 → **Offline** 선택
4. 테스트 완료 후 반드시 **No throttling** 으로 복구

---

## SCENARIO 1 — 오프라인 배너 표시

> **목적**: 네트워크 연결이 끊겼을 때 상단 배너가 즉시 표시되는지 검증

### 1-1. 오프라인 전환 시 배너 표시

**테스트 단계**

1. 앱에 로그인하여 임의 페이지(대시보드 등)에서 확인한다.
2. DevTools → Network → **Offline** 으로 전환한다.

**기대 결과**

- [X] 헤더 바로 아래에 **황색(amber)** 배경 배너가 즉시 표시됨
- [X] 배너에 와이파이 끊김 아이콘(WifiOff)이 표시됨
- [X] 배너 텍스트: **"오프라인 상태입니다. 인터넷 연결을 확인해 주세요."** 가 표시됨
- [X] 화면 너비가 충분한 경우 **"— M월 D일 HH:MM 마지막 연결"** 텍스트가 우측에 표시됨

### 1-2. 온라인 복구 시 배너 사라짐

**테스트 단계**

1. 오프라인 상태에서 배너가 표시된 것을 확인한다.
2. DevTools → Network → **No throttling** 으로 복구한다.

**기대 결과**

- [X] 온라인 복구 즉시 배너가 사라짐
- [X] 페이지 새로고침 없이 자동으로 처리됨

### 1-3. 배너가 메인 콘텐츠를 밀어내는 방식

**테스트 단계**

1. 오프라인 전환 후 페이지 레이아웃을 확인한다.

**기대 결과**

- [X] 배너가 헤더 바로 아래에 위치하고 메인 콘텐츠를 아래로 밀어냄 (overlay 아님)
- [X] 배너 높이만큼 스크롤 영역이 유지됨

### 1-4. 페이지 이동 시 배너 유지

**테스트 단계**

1. 오프라인 상태에서 사이드바의 다른 메뉴(포트폴리오, 매매 일지 등)를 클릭하여 이동한다.

**기대 결과**

- [X] 페이지 이동 후에도 오프라인 배너가 유지됨
- [X] 배너가 모든 페이지에서 동일하게 표시됨

## SCENARIO 2 — AI 채팅 오프라인 차단

> **목적**: 오프라인 상태에서 AI 채팅 기능이 안전하게 차단되는지 검증

### 2-1. 오프라인 시 AI 채팅 안내 메시지

**테스트 단계**

1. 사이드바에서 **AI 분석** 페이지(`/ai-chat`)로 이동한다.
2. DevTools → Network → **Offline** 으로 전환한다.

**기대 결과**

- [X] 입력창 위에 **황색(amber)** 배경의 오프라인 안내 메시지가 표시됨
- [X] 안내 아이콘(WifiOff)과 함께 표시됨
- [X] 안내 문구: **"오프라인 상태에서는 AI 분석을 사용할 수 없습니다. 인터넷 연결 후 이용해 주세요."** 가 표시됨

### 2-2. 입력창 비활성화

**테스트 단계**

1. 오프라인 상태에서 입력창을 클릭하여 타이핑을 시도한다.

**기대 결과**

- [X] 입력창이 비활성화(`disabled`)되어 텍스트 입력이 불가함
- [X] 입력창 placeholder: **"오프라인 상태입니다..."** 로 변경됨
- [X] 입력창 불투명도가 낮아져 비활성화 상태임이 시각적으로 표시됨

### 2-3. 전송 버튼 비활성화

**테스트 단계**

1. 오프라인 상태에서 전송 버튼 상태를 확인한다.

**기대 결과**

- [X] 전송 버튼이 비활성화(`disabled`)됨
- [X] 버튼 클릭이 무효화됨

### 2-4. 온라인 복구 후 AI 채팅 정상화

**테스트 단계**

1. AI 채팅 페이지에서 오프라인 → 온라인으로 복구한다.

**기대 결과**

- [X] 오프라인 안내 메시지가 사라짐
- [X] 입력창이 다시 활성화됨
- [X] 전송 버튼이 다시 활성화됨
- [X] 정상적으로 메시지 전송 가능

### 2-5. 헤더 AI 분석 패널 (ChatPanel) — 오프라인 동작

**테스트 단계**

1. 오프라인 상태에서 헤더의 **AI 분석** 버튼을 클릭하여 채팅 패널을 연다.
2. 메시지 입력을 시도한다.

**기대 결과**

- [X] ChatPanel이 정상적으로 열림
- [X] 메시지 전송 시 네트워크 오류 또는 AI 에러 메시지 표시 (별도 처리)

## SCENARIO 3 — PWA 업데이트 알림

> **목적**: 새 서비스 워커가 감지될 때 업데이트 알림이 표시되는지 검증
> **참고**: 개발 환경(`npm run dev`)에서는 SW가 등록되지 않으므로 빌드 환경에서 테스트한다.

### 3-1. 업데이트 알림 UI 확인 (빌드 환경)

**전제 조건**: `npm run build && npm run preview` 실행 후 테스트

**기대 결과 (코드 검토 기준)**

- [ ] `App.jsx`에 `useRegisterSW`의 `needRefresh` 상태 기반 알림 배너가 구현됨
- [ ] 배너 위치: 화면 하단 중앙 (`fixed bottom-4`)
- [ ] 배너 텍스트: **"새 버전이 있습니다."** 와 **"지금 업데이트"** 버튼 표시
- [ ] **"지금 업데이트"** 클릭 시 `updateServiceWorker(true)` 호출
- [ ] **"✕"** 클릭 시 배너 닫힘

## SCENARIO 4 — 오프라인 상태에서 로컬 기능 정상 동작

> **목적**: 오프라인에서도 인터넷이 필요 없는 기능들이 정상 동작하는지 검증

### 4-1. 매매 일지 기록 (IndexedDB)

**테스트 단계**

1. 오프라인 상태에서 **매매 일지** (`/journal`) 페이지로 이동한다.
2. 새 매매 기록을 입력하고 저장한다.

**기대 결과**

- [X] 오프라인 배너가 표시되지만 매매 일지 입력 폼이 정상 렌더링됨
- [X] 거래 기록 저장 성공 (IndexedDB 로컬 저장)
- [X] 저장 후 목록에 새 기록이 표시됨

### 4-2. 설정 변경 (LocalStorage)

**테스트 단계**

1. 오프라인 상태에서 **설정** (`/settings`) 페이지로 이동한다.
2. 테마를 다크 모드로 변경한다.

**기대 결과**

- [X] 설정 페이지가 정상 렌더링됨
- [X] 테마 변경이 즉시 적용됨 (LocalStorage 저장)

### 4-3. 포트폴리오 조회 (캐시된 데이터)

**테스트 단계**

1. 오전에 온라인에서 포트폴리오 데이터를 로드한다.
2. 오프라인으로 전환 후 포트폴리오(`/portfolio`) 페이지를 새로고침 없이 확인한다.

**기대 결과**

- [X] 오프라인 배너가 표시됨
- [X] 이미 로드된 포트폴리오 데이터(Zustand 메모리 캐시)가 표시됨
- [X] 가격 데이터가 갱신되지 않고 마지막 로드 값이 표시됨 (TanStack Query 캐시 활용)

2. 오프라인으로 전환한다.

**기대 결과**

- [X] 배너가 화면 전체 너비로 표시됨
- [X] **"마지막 연결"** 시간 텍스트는 숨겨짐 (`hidden sm:inline`)
- [X] 아이콘 + 안내 문구만 표시되어 레이아웃이 깨지지 않음
