## SCENARIO 1 — AI 채팅 세션 사이드바 UI

> **목적**: 세션 관리 사이드바가 올바르게 렌더링되는지 검증

### 1-1. 세션 사이드바 기본 렌더링

**테스트 단계**

1. `/ai-chat` 페이지로 이동한다.

**기대 결과**

- [ ] 페이지 좌측에 **"대화 목록"** 레이블이 있는 사이드바가 표시됨
- [ ] 사이드바 우측 상단에 **"+"(Plus)** 아이콘 버튼이 표시됨
- [ ] 사이드바에 세션 목록이 1건 이상 표시됨 (최초 접속 시 "새 대화" 세션 1건 자동 생성)
- [ ] 현재 활성 세션이 **파란색 배경 + 좌측 파란 테두리**로 구분됨

### 1-2. 세션 항목 정보 표시

**테스트 단계**

1. 사이드바의 세션 항목 하나를 확인한다.

**기대 결과**

- [ ] 메시지 아이콘(MessageSquare) + **세션 제목** 이 표시됨
- [ ] 제목 아래에 **"N개 메시지 · M분 전"** 형식의 메타정보가 표시됨
- [ ] 세션 제목이 30자 초과 시 말줄임표(…)로 잘림

### 1-3. 사이드바 토글

**테스트 단계**

1. 메인 채팅 헤더 좌측의 **ChevronLeft(<)** 버튼을 클릭한다.
2. 다시 **ChevronRight(>)** 버튼을 클릭한다.

**기대 결과**

- [ ] 첫 번째 클릭: 사이드바가 **width 0**으로 접혀 숨겨짐 (애니메이션 포함)
- [ ] 두 번째 클릭: 사이드바가 다시 **width 224px(w-56)**으로 펼쳐짐
- [ ] 사이드바 토글 중 채팅 메인 영역이 자연스럽게 확장/축소됨

## SCENARIO 2 — 새 대화 세션 생성

> **목적**: 새 대화 생성 버튼이 올바르게 동작하고 세션이 독립적으로 관리되는지 검증

### 2-1. 사이드바에서 새 대화 생성

**테스트 단계**

1. 사이드바 우측 상단의 **"+"** 버튼을 클릭한다.

**기대 결과**

- [ ] 새 세션 **"새 대화"** 항목이 사이드바 최상단에 추가됨
- [ ] 새 세션이 즉시 활성화됨 (파란색 강조)
- [ ] 메인 채팅 영역이 빈 상태로 초기화됨 (Bot 아이콘 + 빠른 질문 버튼 표시)

### 2-2. 헤더에서 새 대화 생성

**테스트 단계**

1. 메인 헤더 우측의 **"새 대화"** 버튼을 클릭한다.

**기대 결과**

- [ ] 위 2-1과 동일하게 새 세션이 생성되고 활성화됨

### 2-3. 세션 제목 자동 설정

**테스트 단계**

1. 새 대화에서 질문을 입력하고 전송한다. (예: "삼성전자 주가 분석해줘")
2. AI 응답이 완료된 후 사이드바의 세션 제목을 확인한다.

**기대 결과**

- [ ] 세션 제목이 **"새 대화"** 에서 첫 사용자 메시지 기반 제목으로 자동 변경됨
- [ ] 30자 초과 시 말줄임표(…)로 표시됨 (예: "삼성전자 주가 분석해줘")

## SCENARIO 3 — 세션 전환 (이전 대화 이어하기)

> **목적**: 여러 세션 간 전환 시 각 세션의 메시지가 독립적으로 유지되는지 검증

### 3-1. 복수 세션 생성 후 전환

**테스트 단계**

1. **세션 A**: 새 대화에서 "내 매매 패턴 분석해줘" 입력 후 AI 응답 대기
2. **"새 대화"** 버튼으로 새 세션 생성
3. **세션 B**: "삼성전자 분석해줘" 입력 후 AI 응답 대기
4. 사이드바에서 세션 A를 클릭한다.

**기대 결과**

- [ ] 세션 A의 메시지 목록("내 매매 패턴 분석해줘" + AI 응답)이 표시됨
- [ ] 세션 B의 내용이 표시되지 않음 (세션 간 독립)
- [ ] 활성 세션 표시가 세션 A로 변경됨

### 3-2. 세션 전환 후 대화 이어하기

**테스트 단계**

1. 세션 A로 전환된 상태에서 추가 질문을 입력한다.

**기대 결과**

- [ ] 새 메시지가 세션 A의 기존 대화에 이어서 추가됨
- [ ] 세션 B는 영향받지 않음

### 3-3. 세션 날짜 표시 검증

**테스트 단계**

1. 사이드바의 세션 항목들의 날짜 표시를 확인한다.

**기대 결과**

- [ ] 방금 전 메시지: **"N분 전"** 형식으로 표시됨
- [ ] 1시간 이상: **"N시간 전"** 형식으로 표시됨
- [ ] 하루 이상: **"M/D"** 형식으로 표시됨 (예: `4/19`)

## SCENARIO 4 — 세션 초기화 및 삭제

> **목적**: 현재 세션 초기화 및 삭제 동작을 검증

### 4-1. 단일 세션 초기화 (메시지만 삭제)

**전제 조건**: 세션이 1개만 존재하는 상태

**테스트 단계**

1. 대화가 있는 상태에서 헤더의 **"초기화"** 버튼을 클릭한다.

**기대 결과**

- [ ] 현재 세션의 메시지 목록이 모두 삭제됨
- [ ] 세션 자체는 유지됨 (사이드바에 "새 대화" 세션 1건 잔류)
- [ ] 채팅 영역이 빈 상태로 초기화됨 (Bot 아이콘 + 빠른 질문 버튼)

### 4-2. 복수 세션 중 현재 세션 삭제

**전제 조건**: 세션이 2개 이상 존재하는 상태

**테스트 단계**

1. 삭제하려는 세션을 활성화 상태로 전환한다.
2. 헤더의 **"초기화"** 버튼을 클릭한다.

**기대 결과**

- [ ] 현재 활성 세션이 삭제됨
- [ ] 남은 세션 중 가장 최근 세션이 자동으로 활성화됨
- [ ] 사이드바의 세션 수가 1개 감소함

## SCENARIO 5 — IndexedDB 세션 저장 및 복원

> **목적**: 대화 세션이 IndexedDB에 저장되고 새로고침 후 복원되는지 검증

### 5-1. 메시지 전송 후 IndexedDB 저장 확인

**테스트 단계**

1. 새 대화에서 AI에게 질문을 전송한다.
2. AI 응답이 완료된 후 브라우저 DevTools → Application → IndexedDB → **PortfolioHub** → **chatHistory** 테이블을 확인한다.

**기대 결과**

- [ ] `chatHistory` 테이블에 레코드가 1건 이상 존재함
- [ ] 레코드에 `sessionId`, `userId`, `agentType`, `messages`, `updatedAt` 필드가 존재함
- [ ] `messages` 배열에 방금 전송한 사용자 메시지와 AI 응답이 포함됨

### 5-2. 페이지 새로고침 후 세션 복원

**테스트 단계**

1. 대화가 있는 상태에서 페이지를 새로고침(F5)한다.

**기대 결과**

- [ ] 새로고침 후 이전 세션 목록이 사이드바에 표시됨 (LocalStorage 복원)
- [ ] 마지막 활성 세션의 메시지가 메인 영역에 표시됨

## SCENARIO 6 — 10턴 초과 대화 메모리 압축

> **참고**: 긴 테스트이므로 간소화하여 진행 가능

### 6-1. 대화 압축 확인 (DevTools 활용)

**테스트 단계**

1. 브라우저 DevTools → **Network** 탭을 열어 `/api/claude` 요청을 모니터링한다.
2. 동일 세션에서 연속으로 11번 이상 메시지를 전송한다.

**기대 결과**

- [ ] 11번째 이후 요청의 `messages` 배열 첫 번째 항목이 `"[이전 대화 요약]"` 텍스트를 포함함
- [ ] 두 번째 항목이 `"이전 대화 내용을 확인했습니다."` 형태의 AI 응답임
- [ ] 최근 4개 메시지가 요약 뒤에 이어서 포함됨
- [ ] AI 응답이 이전 대화 맥락을 유지하며 정상적으로 반환됨

## SCENARIO 7 — 주간 리포트 알림 토스트

> **목적**: 로그인 후 이번 주 주간 리포트가 있을 때 토스트 알림이 표시되는지 검증
> **참고**: 주간 리포트가 DB에 저장된 상태가 전제 조건

### 7-1. 주간 리포트 알림 표시 (DB 데이터 있는 경우)

**전제 조건**: Reports 페이지에서 먼저 주간 리포트를 1회 생성 + 저장한 상태

**테스트 단계**

1. Reports 페이지에서 "AI 인사이트 생성" 버튼으로 리포트를 생성한다. (저장 기능이 연동된 경우)
2. 페이지를 완전히 새로고침하거나, 로그아웃 후 재로그인한다.

**기대 결과**

- [ ] 로그인 후 약 2초 뒤 화면 우측 하단에 **📊 주간 리포트 알림 토스트**가 표시됨
- [ ] 토스트 제목: **"📊 지난 주간 리포트가 있습니다."**
- [ ] 토스트 설명: 리포트 제목 (예: `2026년 4월 16주차 주간 리포트`)
- [ ] **"확인하기"** 액션 버튼이 표시됨
- [ ] "확인하기" 클릭 시 `/reports` 페이지로 이동됨
- [ ] 토스트가 약 8초 후 자동으로 사라짐

### 7-2. 주간 리포트 없을 때 알림 미표시

**전제 조건**: 이번 주에 생성된 주간 리포트가 없는 상태

**테스트 단계**

1. 로그인 후 5초간 기다린다.

**기대 결과**

- [ ] 주간 리포트 관련 토스트가 표시되지 않음 (다른 토스트에 영향 없음)

## SCENARIO 8 — 정기 리포트 생성 및 저장

> **목적**: Reports 페이지에서 주간/월간 리포트 템플릿을 사용해 리포트를 생성하고 저장하는지 검증

### 8-1. Reports 페이지 AI 인사이트 생성

**테스트 단계**

1. 좌측 사이드바에서 **리포트** 페이지(`/reports`)로 이동한다.
2. **"AI 인사이트"** 탭을 클릭한다.
3. **"AI 인사이트 생성"** 버튼을 클릭한다.

**기대 결과**

- [ ] 버튼 클릭 후 로딩 상태가 표시됨 (스피너 또는 텍스트 변경)
- [ ] AI 응답이 마크다운 형식으로 렌더링됨
- [ ] 응답 내용에 성과 요약, 종목 분석, 인사이트 섹션이 포함됨
- [ ] 응답 끝에 면책 문구가 포함됨

### 8-2. 면책 문구 다양화 확인

**테스트 단계**

1. AI 인사이트를 3회 이상 생성한다. (생성 버튼 반복 클릭)
2. 각 응답 끝의 면책 문구를 비교한다.

**기대 결과**

- [ ] 3회 중 적어도 1회는 다른 표현의 면책 문구가 표시됨
- [ ] 아래 4종 중 하나가 표시됨:
  - `"이 분석은 참고용이며 투자 결정의 책임은 본인에게 있습니다."`
  - `"본 내용은 투자 참고 자료이며, 모든 투자 결정과 그 결과는 투자자 본인의 책임입니다."`
  - `"위 분석은 데이터 기반 참고 정보로, 실제 투자에 따른 손익은 투자자 본인이 부담합니다."`
  - `"이 내용은 투자 심리 코칭 목적이며, 특정 종목에 대한 투자 권유가 아닙니다. 투자 결정은 본인이 직접 판단하세요."`

## SCENARIO 10 — JournalCoach 컨텍스트 압축 (200건 초과)

> **목적**: 매매 일지가 200건 초과 시 압축 컨텍스트가 사용되는지 확인
> **참고**: 실제 200건 데이터를 만들기 어려우므로 코드 검토로 대체 가능

### 10-1. 코드 검토 기반 확인

**테스트 단계**

1. 브라우저 DevTools → Sources 탭 → `journalCoachAgent.js` 파일을 찾아 열어본다.

**기대 결과 (코드 검토 기준)**

- [ ] `buildCompressedJournalContext` 함수가 존재함
- [ ] 함수 내부에서 `entries.length <= 200` 조건으로 일반/압축 컨텍스트를 분기함
- [ ] 압축 시 전체 통계 요약 + 최근 20건만 포함하는 로직이 구현됨

**기대 결과**

- [ ] 200건 초과 시 `systemPrompt`에 `"[매매 일지 압축 요약 — 전체 데이터 기반]"` 텍스트 포함
- [ ] 최근 20건만 상세 기록, 전체 통계는 요약 형태로 포함됨
