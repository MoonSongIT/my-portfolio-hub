### 7.4-B. `upsertSync()` 구현

> 파일: `src/utils/stockMasterDb.js` (Sprint 7.1 연장)

- [ ] `upsertSync(exchange, remoteRows)` 함수
  - 1단계: 기존 local rows 조회 (`[category+exchange]` 인덱스)
  - 2단계: `Set` 기반 diff — added / removed / common
  - 3단계: common 중 `__isDirty()` 판별 (name/sector/corpCode/type 비교)
  - 4단계: `mergeRow(local, remote)` — firstSeenAt/isCustom 보존
  - 5단계: Dexie 트랜잭션 안에서 `bulkPut(added+changed)` + `modify(removed → isActive=false)`
  - 6단계: `appendSyncLog()`
- [ ] 반환 형식: `{ added: N, changed: N, removed: N, durationMs }`
- [ ] `isCustom=true` row는 removed 후보에서 제외 (사용자 데이터 보호)
