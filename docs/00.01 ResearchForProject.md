## 1. 기술 스택 심층 분석

### 1.1 React + Vite 금융 대시보드 베스트 프랙티스

**Vite 설정 최적화**
- **Vite 6.x** 사용 권장, React 플러그인은 `@vitejs/plugin-react-swc` (SWC 기반, 빌드 속도 20~30% 향상)
- 금융 대시보드는 **코드 스플리팅**이 핵심: `React.lazy()` + `Suspense`로 페이지별 분할
- `vite.config.js`에서 `build.rollupOptions.output.manualChunks`로 차트 라이브러리를 별도 청크로 분리

**성능 최적화 패턴**
- `React.memo`로 종목 카드/테이블 행 불필요한 리렌더 방지
- `useMemo`/`useCallback`으로 수익률 계산 등 비용 큰 연산 캐싱
- 대량 종목 목록은 **가상화 필수**: `@tanstack/react-virtual` v3.x
- React 18의 `useDeferredValue`로 실시간 가격 업데이트 시 UI 블로킹 방지

**추천 패키지**
```
@vitejs/plugin-react-swc: ^4.x
@tanstack/react-virtual: ^3.x
react: ^18.3.x
react-router-dom: ^6.x
```

### 1.4 상태관리: Zustand 검증

| 라이브러리 | 번들 크기 | 보일러플레이트 | 금융 앱 적합도 |
|-----------|----------|-------------|-------------|
| **Zustand** | ~1KB | 최소 | 높음 |
| Redux Toolkit | ~11KB | 중간 | 높음 (대규모 팀) |
| Jotai | ~3KB | 최소 | 중간 |
| **Zustand + Immer** | ~4KB | 최소 | **매우 높음** |

**Zustand 금융 앱 패턴**
```javascript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import { immer } from 'zustand/middleware/immer';

const usePortfolioStore = create(
  persist(
    immer((set) => ({
      holdings: [],
      addHolding: (stock) => set((state) => { state.holdings.push(stock); }),
      updatePrice: (ticker, price) => set((state) => {
        const h = state.holdings.find(h => h.ticker === ticker);
        if (h) h.current_price = price;
      }),
    })),
    { name: 'portfolio-storage' }
  )
);
```

**결론**: Zustand 유지 + `immer` 미들웨어 추가 권장

### 1.7 API 키 보안 전략

> **핵심 원칙**: 프론트엔드에 노출되는 `VITE_` 접두사 환경변수는 빌드 번들에 포함되어 사용자에게 공개된다.

| 레벨 | 방법 | 적합 단계 |
|------|------|----------|
| **레벨 1** | `.env` + `.gitignore`, 무료 API 키만 사용 | Phase 1 MVP |
| **레벨 2** | Vite dev proxy / Vercel Serverless Functions | Phase 3~4 |
| **레벨 3** | 전용 백엔드 + Rate limiting + 키 로테이션 | 프로덕션 |

**중요 경고**: `VITE_ANTHROPIC_API_KEY`는 종량제이므로 **반드시 서버사이드 프록시를 통해 호출**해야 함. 브라우저 개발자 도구로 추출 가능.

```javascript
// Vercel Serverless Function 예시: /api/claude.js
export default async function handler(req, res) {
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    headers: {
      'x-api-key': process.env.ANTHROPIC_API_KEY, // VITE_ 접두사 없음 = 서버 전용
      'anthropic-version': '2023-06-01',
    },
    body: JSON.stringify(req.body),
  });
  res.json(await response.json());
}
```

VitePWA({

## 2. 경쟁 서비스 조사

### 2.1 글로벌 포트폴리오 관리 서비스

#### Yahoo Finance
- **핵심 기능**: 실시간 시세, 포트폴리오 추적, 뉴스 피드, 스크리너, 재무제표 조회
- **UI/UX**: 대시보드 중심 직관적 레이아웃, 커스터마이즈 가능한 워치리스트
- **가격**: 무료 (Plus: 월 $25~$50)
- **차별점**: 가장 방대한 무료 주가 데이터, 비공식 API 생태계 풍부
- **시사점**: 본 프로젝트의 주가 데이터 소스로 활용 예정

#### Google Finance
- **핵심 기능**: 포트폴리오 추적, 시장 동향, Google Sheets 연동
- **UI/UX**: 미니멀 구글 스타일, 검색 중심 UX
- **가격**: 완전 무료
- **차별점**: `GOOGLEFINANCE()` 함수로 스프레드시트 연동 강력
- **시사점**: 미니멀 UI 디자인 참고

#### Seeking Alpha
- **핵심 기능**: 종목 분석 리포트, 퀀트 등급, 배당 추적, 포트폴리오 트래커
- **UI/UX**: 콘텐츠 중심 레이아웃, 커뮤니티 분석 글이 핵심
- **가격**: 무료(제한) / Premium $24.99/월 / Pro $58.33/월
- **차별점**: 크라우드소싱 리서치, Quant Rating 시스템
- **시사점**: AI 분석 리포트의 형식/구성 참고

#### Stock Rover
- **핵심 기능**: 고급 스크리너, 포트폴리오 분석, 상관관계 분석, 리밸런싱 도구
- **UI/UX**: 데이터 밀도 높은 대시보드, 파워유저 지향
- **가격**: 무료(제한) / Essentials $7.99/월 / Premium $17.99/월
- **차별점**: 미국 내 가장 강력한 펀더멘털 스크리너, 10년 이상 재무 데이터
- **시사점**: 종목 스크리너 기능 설계 참고

#### Morningstar
- **핵심 기능**: 펀드/ETF 등급, X-Ray 포트폴리오 분석, 독립 리서치 리포트
- **가격**: 무료(제한) / Premium ~$35/월
- **차별점**: 독자 별점 등급 시스템(Star Rating), 펀드/ETF 분석 업계 표준
- **시사점**: ETF 분석 지표 체계 참고

#### Sharesight
- **핵심 기능**: 포트폴리오 추적, 배당 추적, 세금 리포트, 다중 통화 지원
- **UI/UX**: 깔끔한 모던 UI, 리포트 시각화 우수
- **가격**: 무료(10종목) / Investor $15/월 / Expert $24/월
- **차별점**: 다국가 세금 리포트, 배당 재투자 자동 추적
- **시사점**: 리포트 시각화, 다중 통화 처리 참고

#### Empower (구 Personal Capital)
- **핵심 기능**: 순자산 추적, 투자 체크업, 은퇴 플래너, 수수료 분석기

- **시사점**: 종합 대시보드 설계 참고

                    ↓
         [응답 생성 + 면책 조항]

#### RAG for Financial Data

```
[데이터 소스]                    [벡터 DB]
- SEC 10-K/10-Q 공시     →    Pinecone / Weaviate / Chroma
- 어닝콜 트랜스크립트    →    임베딩: text-embedding-3-large

[질의 처리]
1. 질의 분해 (Multi-hop reasoning)
2. 하이브리드 검색 (키워드 + 시맨틱)
3. 리랭킹 (Cohere Rerank, Cross-encoder)
4. 컨텍스트 윈도우 최적화
5. LLM 생성 + 수치 검증
```
