## 2. 통합 테스트 (수동 — 로컬 서버)

> 사전 조건: `npm run dev` 로 Vite 개발 서버 실행

### 2-1. 서버 API 응답 검증

```powershell
# KOSPI — rows ≥ 800 기대
Invoke-RestMethod "http://localhost:5173/api/stock-master?exchange=KOSPI" | Select-Object count, exchange

# DOMESTIC 전체 — 4개 exchange 포함
Invoke-RestMethod "http://localhost:5173/api/stock-master?category=DOMESTIC" | Select-Object -ExpandProperty rows | Group-Object exchange | Select-Object Name, Count

# manifest — rows 없이 카운트만
Invoke-RestMethod "http://localhost:5173/api/stock-master/manifest"
```

**기대 결과**

| 시장 | 최소 rows |
|------|-----------|
| KOSPI | 800 |
| KOSDAQ | 1,000 |
| KRX_ETF | 600 |
| NXT | 47 |
| NASDAQ | 3,000 |
| NYSE | 2,000 |
| US_ETF | 500 |

### 2-2. 2회 연속 syncAll 검증

1. 설정 페이지 → [전체 업데이트] 실행 (1회차)
2. 완료 후 바로 [전체 업데이트] 재실행 (2회차)
3. **기대**: 2회차 `변경 종목` 5개 미만

### 2-3. DART 키 미설정 검증

1. `.env` 에서 `DART_API_KEY` 주석 처리
2. `?exchange=KOSPI` 요청
3. **기대**: HTTP 200 + `errors` 배열에 DART 오류 포함, UI 에러 배지 표시

### 2-4. 트랜잭션 롤백 검증

1. DevTools → Application → IndexedDB → `StockMasterDB` → `stocks` 현재 count 기록
2. 설정 → [국내 업데이트] 실행 중 네트워크 끊기
3. **기대**: 완료된 부분만 저장, DB 일관성 유지 (count 변화 없거나 완전 저장)

## 3. E2E 시나리오 (Playwright — 수동 절차)

> Playwright 미설치 시 브라우저에서 직접 수행

### M1 — 신규 사용자 전체 업데이트

```
경로: 설정 → 종목 마스터 DB
조작: [전체 업데이트] 클릭
검증:
  - 진행바가 exchange 순서로 이동 (KOSPI → KOSDAQ → KRX_ETF → ...)
  - 완료 토스트 "전체 업데이트 완료" 표시
  - 국내/해외 각 카드의 카운트 > 0
  - DevTools IDB stocks 레코드 5,000 개 이상
```

### M2 — 기존 LocalStorage 사용자 마이그레이션

```
사전: localStorage에 'stock-db-v1' 키 존재
조작: 앱 첫 로드
검증:
  - IDB StockMasterDB 에 데이터 이전됨
  - 'stock-db-v1-migrated' 플래그 설정됨
  - 재로드 시 재마이그레이션 발생 안 함
```

### M3 — 동일 상태 재 sync

```
사전: 전체 업데이트 완료 후
조작: 설정 → [증분 동기화]
검증: "변경된 시장이 없습니다" 토스트 표시
```

### M4 — 종목 검색 (한글)

```
경로: 종목 탐색 페이지
조작: "삼성" 입력
검증:
  - 005930 삼성전자 자동완성 표시
  - Network 탭: 외부 API 요청 없음 (IDB 로컬)
  - 응답 시간 < 50ms
```

### M5 — 종목 검색 (영문 티커)

```
조작: "AAPL" 입력
검증: NASDAQ AAPL 결과 표시
```

### M6 — 오프라인 모드 검색

```
조작: DevTools → Network → Offline 설정 후 "삼성전자" 검색
검증: 검색 결과 정상 표시 (IDB 기반)
```

### M7 — 포트폴리오 미검증 티커 경고

```
경로: 포트폴리오 → 종목 추가
사전: 종목 DB 동기화 완료 상태
조작: 티커 입력란에 "ZZZZZ" 입력 후 Tab
검증: 노란 경고 배너 "검증되지 않은 종목" 표시
```

### M8 — NXT 병합 확인

```
Invoke-RestMethod "http://localhost:5173/api/stock-master?exchange=KOSPI" |
  Select-Object -ExpandProperty rows |
  Where-Object { $_.ticker -eq '005930' } |
  Select-Object tradableOn

# 기대: tradableOn 에 'KOSPI', 'NXT' 포함
```

### M9 — 커스텀 종목 보호

```
경로: 설정 → 커스텀 종목 추가 → 임의 종목 추가
조작: [전체 업데이트] 실행
검증: 커스텀 종목이 isActive=true 유지 (제거 안됨)
```

### M10 — 상장폐지 soft delete

```
조작: IDB 에서 임의 row isActive=false 로 수동 수정
검증:
  - 종목 탐색 검색에서 해당 종목 미표시
  - upsertSync 재실행 시 해당 row 복원됨
```

### M11 — AI 채팅 티커 해석

```
경로: AI 분석 → ResearchAgent
입력: "삼성전자 분석해줘"
검증: 티커 005930 KOSPI 자동 해석 → 분석 리포트 생성
```

### M12 — 증분 동기화 카운트 변경

```
조작: IDB 에서 임의 row 삭제 후 [증분 동기화]
검증: 해당 exchange 만 재sync, 나머지 skip
```

## 4. 마이그레이션 시나리오 검증

### 4-1. 정상 마이그레이션

```javascript
// DevTools Console에서 실행
localStorage.setItem('stock-db-v1', JSON.stringify({
  stocksByMarket: [
    { ticker: 'TEST01', name: '테스트종목', market: 'KRX', exchange: 'KOSPI', type: 'EQUITY', currency: 'KRW' }
  ],
  customStocks: []
}))
// 페이지 새로고침 후
// IDB stocks 에 TEST01 이전 확인
// localStorage['stock-db-v1-migrated'] 존재 확인
```

### 4-2. 중복 마이그레이션 방지

```javascript
// 위 시나리오 후 재로드
// 'stock-db-v1-migrated' 키가 있으면 마이그레이션 스킵 확인
```

### 4-3. 마이그레이션 실패 처리

```javascript
// 잘못된 형식의 데이터 주입
localStorage.setItem('stock-db-v1', 'invalid_json')
// 페이지 재로드 → 에러 토스트 + 설정 페이지 유도 안내 확인
```

## 5. 보안 체크리스트

- [ ] `DART_API_KEY` — `.env` 서버 전용, `VITE_` 접두사 없음 확인
- [ ] NASDAQ Trader 요청 — `User-Agent` 헤더 포함 (차단 방지)
- [ ] Naver ETF 요청 — `Referer: https://finance.naver.com` 헤더 유지
- [ ] 마스터 DB 데이터 — 공개 정보만 (PII 없음)
- [ ] `isCustom=true` 사용자 데이터 — 서버 sync로 덮어쓰지 않음
