## 2. 신규 기능 개요

### 기능 A — 계좌별 입출금 & 투자 가능 금액 관리

- 계좌별로 입금/출금 내역을 기록하고 투자 가능 금액을 자동 계산
- 매수 시 투자 가능 금액에서 자동 차감, 매도 시 자동 증가
- 투자 가능 금액 = 총 입금 − 총 출금 − 현재 보유 종목 매수 원가 합산

### 기능 B — 종목별 1일 손익 기록 & 등락 그래프

- 종가 기준으로 매일 종목별 손익 스냅샷 저장
- 투자 시점(최초 매수일)부터 현재까지 누적 수익률 그래프 제공
- 복수 종목 오버레이 및 벤치마크(KOSPI/S&P500) 비교

## 3. 데이터 모델 설계

### 3-1. 입출금 내역 (cashFlowStore)

```js
{
  id: UUID,
  accountId: string,
  type: 'deposit' | 'withdrawal',   // 입금 | 출금
  amount: number,
  currency: 'KRW' | 'USD',
  date: string,                      // YYYY-MM-DD
  memo: string,
  createdAt: ISO8601
}
```

**파생 계산값 (셀렉터)**
```
투자 가능 금액(accountId)
  = 총 입금 합산
  - 총 출금 합산
  - journalStore.computeHoldings(accountId)의 totalCost 합산
```

### 3-2. 일별 손익 스냅샷 (dailyPnlStore)

```js
{
  id: UUID,
  date: string,              // YYYY-MM-DD
  ticker: string,
  name: string,
  accountId: string,
  closePrice: number,        // 해당일 종가
  quantity: number,          // 해당일 보유 수량
  avgBuyPrice: number,       // 평균 매수가
  investedAt: string,        // 최초 매수일 (투자 시점)
  dailyPnl: number,          // 전일 대비 1일 손익 (closePrice - prevClosePrice) × quantity
  dailyPnlRate: number,      // 1일 손익률 (%)
  cumulativePnl: number,     // 투자 시점 대비 누적 손익
  cumulativePnlRate: number, // 투자 시점 대비 누적 수익률 (%)
  createdAt: ISO8601
}
```

**저장 정책**
- IndexedDB (`daily_pnl` 테이블) 우선, localStorage 백업
- 보존 기간: 최소 2년치 (약 500거래일 × 보유 종목 수)

### 3-3. accountStore 확장

기존 계좌 모델에 `initialBalance` 필드 추가:
```js
{
  id: UUID,
  name: string,
  broker: string,
  type: 'GENERAL' | 'ISA' | 'IRP' | 'PENSION' | 'ETC',
  currency: 'KRW' | 'USD',
  initialBalance: number,   // ← 신규: 계좌 개설 시 초기 잔고
  memo: string,
  createdAt: ISO8601
}
```

## 4. 아키텍처 & 데이터 흐름

### 4-1. 입출금 흐름

```
사용자: 입금/출금 입력 (CashFlowModal)
  ↓
cashFlowStore.addCashFlow()
  ├→ localStorage 저장
  └→ IndexedDB 저장 (비동기)

  ↓ (파생 계산)
cashFlowStore.getAvailableCash(accountId)
  = 총입금 - 총출금 - journalStore.computeHoldings(accountId).totalCost
  ↓
AvailableCashCard 표시
```

### 4-2. 매수/매도 ↔ 현금 흐름 자동 연동

```
journalStore.addEntry({ action: 'buy', price, quantity, accountId })
  ├→ 기존: entries 배열에 추가
  └→ 신규: cashFlowStore.addAutoFlow({ type: 'withdrawal', amount: price×quantity })

journalStore.addEntry({ action: 'sell', price, quantity, accountId })
  ├→ 기존: entries 배열에 추가
  └→ 신규: cashFlowStore.addAutoFlow({ type: 'deposit', amount: price×quantity })
```

> `addAutoFlow`: 자동 연동 플래그(`isAuto: true`)로 수동 입출금과 구분

### 4-3. 일별 손익 수집 파이프라인

```
앱 시작 / Dashboard 진입
  ↓
dailyPnlService.snapshotToday()
  ├→ journalStore.computeAllHoldings() — 현재 보유 종목 조회
  ├→ stockApi.fetchDailyClose(tickers) — 오늘 종가 조회
  ├→ 전일 스냅샷과 비교 → dailyPnl 계산
  └→ dailyPnlStore.saveSnapshots() — IndexedDB 저장

종목 신규 매수 시
  ↓
dailyPnlService.backfillHistory(ticker, fromDate: investedAt)
  ├→ stockApi.fetchHistoricalClose(ticker, from, to) — 과거 종가 일괄 조회
  └→ dailyPnlStore.bulkSave() — 과거 스냅샷 일괄 저장
```
