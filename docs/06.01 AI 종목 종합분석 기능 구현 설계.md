# AI 종목 종합분석 기능 구현 설계

> **목적**: 사용자가 "현재 삼성전자를 분석해줘"라고 질문했을 때
> **현재 주가 + 뉴스 + 실적 + 공시자료**를 조합적으로 수집·조사하여
> Claude API가 종합 리포트를 생성하는 기능의 구현 방법을 정리한 문서입니다.
>
> 작성일: 2026-04-20
> 대상 파일: `src/agents/researchAgent.js`, `src/api/claudeApi.js`, `src/api/stockApi.js`, `vite.config.js`, `server/index.js`

## 1. 현재 상태 진단 (Gap Analysis)

### 1.1 이미 구현된 것 ✅

| 영역 | 파일 | 기능 |
|------|------|------|
| 현재가/52주 고저/시총 | `src/api/stockApi.js` → `fetchQuote()` | Yahoo v8 / Naver 경유 실시간 시세 |
| 재무 지표 (PER/PBR/ROE 등) | `src/api/stockApi.js` → `fetchProfile()` | yahoo-finance2 quoteSummary (`summaryProfile`, `summaryDetail`, `defaultKeyStatistics`, `financialData`) |
| 가격 히스토리 | `src/api/stockApi.js` → `fetchHistory()` | Yahoo/Naver 일봉 데이터 |
| 종목 검색 (한글/영문) | `src/api/stockApi.js` → `fetchSearch()` | 로컬 DB + Yahoo Finance search |
| Research 시스템 프롬프트 | `src/agents/researchAgent.js` | 4개 프레임워크(기본·재무·기술·리스크) |
| Claude API 프록시 | `vite.config.js` / `server/index.js` | `/api/claude` 경유 서버사이드 호출 |

### 1.2 비어 있는 것 ❌

1. **뉴스 수집**: 어느 API 경로에서도 뉴스를 불러오지 않음.
2. **공시자료**: 한국 DART / 미국 EDGAR(SEC) 연동 없음.
3. **최근 4분기 실적 트렌드**: `financialData`에는 단일 분기만 포함 → `earningsHistory` / `earningsTrend` 모듈 미요청.
4. **기술적 지표(RSI/MACD/이평선)**: `fetchHistory` 결과에서 별도 계산 안 함.
5. **`buildResearchContext`의 입력 범위가 너무 좁음**: `stockData` 단일 객체만 받아 뉴스·공시를 넘길 자리가 없음.
6. **Research 호출 시 실제 데이터를 프리패치하는 파이프라인 부재**: 현재 `sendToAgent({stockData})`는 상위에서 `stockData`를 만들어 주입해야 하는데, 이를 조립하는 오케스트레이션 코드가 없음.

## 2. 아키텍처 선택지 비교

Claude가 외부 데이터를 "알고 답하게" 만드는 방법은 크게 두 가지입니다.

### 2.1 A안 — **프리패치 컨텍스트 주입** (권장: 1단계 MVP)

```
사용자 질문
   ↓
티커 추출 (ex. 삼성전자 → 005930, KRX)
   ↓
병렬 페치: [시세 | 프로필 | 재무분기 | 뉴스 | 공시 | 기술지표]
   ↓
buildResearchContext({stockData, news, disclosures, earnings, tech})
   ↓
Claude (RESEARCH_PROMPT + 컨텍스트) → 리포트 생성
```

**장점**
- 기존 프록시(`/api/claude`) 구조 그대로 사용 가능 (툴 없음).
- 병렬 fetch → 지연시간 최소화.
- 응답 결정성이 높고 디버깅 쉬움.

**단점**
- 질문에서 티커를 추출하는 전처리가 필요.
- 토큰 사용량이 일정하게 큼(불필요한 데이터도 항상 동봉).

### 2.2 B안 — **Claude Tool Use** (2단계 확장)

Claude Messages API의 `tools` 파라미터로 "내부 MCP처럼" 함수 호출:

```json
{
  "tools": [
    { "name": "get_quote",        "input_schema": {...} },
    { "name": "get_financials",   "input_schema": {...} },
    { "name": "get_news",         "input_schema": {...} },
    { "name": "get_disclosures",  "input_schema": {...} }
  ],
  "messages": [{"role":"user","content":"삼성전자 분석해줘"}]
}
```

**장점**
- Claude가 필요한 도구만 선택적으로 호출 → 토큰 절약.
- 모호한 질문(ex. "삼성전자 요즘 이슈 있어?")도 뉴스만 집중 조회 가능.

**단점**
- 다중 라운드 교환 필요 (`tool_use` → `tool_result` → 최종 답변).
- 프록시 서버에서 tool 핸들러 라우팅·실행·재전송 로직 구현 필요.
- 지연시간 1라운드 더 소요.

### 2.3 C안 — **Anthropic Web Search 도구** (선택적)

Claude 2024~2025 기준 `web_search` 내장 도구를 지원합니다. 뉴스/공시 실시간 조회를 외부 API 없이 해결할 수 있지만 **한국어 공시(DART)·실적 구체 수치는 여전히 전용 API가 정확**하므로 "뉴스 보강" 용도로만 추천.

### 2.4 결정

> **1단계(이번 스프린트): A안 프리패치**
> **2단계(후속): B안으로 리팩토링 + C안을 뉴스 백업으로 활용**

## 3. 데이터 소스 상세

### 3.1 현재 주가 (이미 구현됨)
- **사용 함수**: `fetchQuote(ticker, market)` — `src/api/stockApi.js:25`
- **반환 필드**: `currentPrice`, `previousClose`, `changePercent`, `volume`, `fiftyTwoWeekHigh/Low`, `marketCap`

### 3.2 재무 지표 (부분 구현 → 보강 필요)
- **사용 함수**: `fetchProfile(ticker, market)` — `src/api/stockApi.js:199`
- **현재 모듈**: `summaryProfile`, `summaryDetail`, `defaultKeyStatistics`, `financialData`, `price`
- **추가 권장 모듈**:
  - `earningsHistory` — 최근 4분기 EPS 서프라이즈
  - `earningsTrend` — 매출·EPS 예상치 트렌드
  - `incomeStatementHistoryQuarterly` — 분기 손익
  - `recommendationTrend` — 애널리스트 매수/매도 분포

### 3.3 뉴스 — **신규 통합 필요**

| 소스 | 장점 | 단점 | 권장도 |
|------|------|------|--------|
| **Yahoo Finance `/v1/finance/search?newsCount=N`** | 이미 프록시 통과 (`/api/yahoo`), 글로벌 커버리지 | 한국 로컬 뉴스는 얕음 | ★★★ |
| **Naver 금융 뉴스 크롤링** | 한국어 뉴스 풍부 | HTML 파싱 필요, 약관 리스크 | ★★ |
| **NewsAPI.org** | 정형 JSON, 다국어 | 유료 플랜 필요 | ★★ |
| **Claude Web Search Tool** | 키 불필요, 최신 | 환각 가능성, 인용 검증 필요 | ★★ |

**권장**: Yahoo 뉴스 + (한국 종목 한정) Naver 뉴스 프록시 병용.

### 3.4 공시자료 — **신규 통합 필요**

#### 한국 종목 → **DART OpenAPI** (금융감독원 전자공시)
- 엔드포인트: `https://opendart.fss.or.kr/api/list.json`
- 필수 파라미터: `crtfc_key`, `corp_code`, `bgn_de`, `end_de`
- 주의: 티커(005930) ≠ 고유번호(corp_code). 사전에 `corpCode.xml`로 매핑 테이블 다운로드 필요.
- 인증: 무료 키 발급(개인) — 서버사이드 보관.

#### 미국 종목 → **SEC EDGAR API**
- 엔드포인트: `https://data.sec.gov/submissions/CIK{cik}.json`
- 티커→CIK 매핑: `https://www.sec.gov/files/company_tickers.json`
- 인증 불필요, `User-Agent` 헤더 필수(개인 이메일).

### 3.5 기술적 지표 — **클라이언트 계산**
이미 있는 `fetchHistory()` 결과를 가지고 프론트/서버 어디서든 계산 가능:
- **이동평균**: `MA(20/60/120) = sum(close, N)/N`
- **RSI(14)**: 평균상승/평균하락 비율
- **MACD(12,26,9)**: EMA 차이 + signal line
- **지지/저항**: 최근 N일 최고/최저 & 피봇

별도 패키지로 **`technicalindicators` (npm)** 사용 시 구현량이 10줄 내로 줄어듭니다.

### 4.3 researchOrchestrator (의사코드)

```js
// src/services/researchOrchestrator.js  — 설계 초안 (실제 코드 아님)
async function assembleResearchBundle({ ticker, market }) {
  const [quote, profile, history, news, disclosures] = await Promise.allSettled([
    fetchQuote(ticker, market),
    fetchProfile(ticker, market),
    fetchHistory(ticker, market, '6mo', '1d'),
    fetchNews(ticker, market),            // 신규
    fetchDisclosures(ticker, market),     // 신규
  ])

  const tech = computeIndicators(history.value || [])  // MA/RSI/MACD

  return {
    stockData:  quote.value,
    profile:    profile.value,
    tech,
    news:       news.value?.slice(0, 5) ?? [],
    disclosures: disclosures.value?.slice(0, 5) ?? [],
  }
}
```

### 4.6 서버 프록시 추가 (DART / EDGAR)

- **DART**: API 키가 민감 → **반드시 `server/index.js`에 새 라우트**로 추가.
- **EDGAR**: 키는 필요 없지만 User-Agent 규정 준수 필요 → 프록시에서 헤더 주입 편함.

```text
GET /api/dart/list?ticker=005930&days=30
GET /api/edgar/filings?ticker=AAPL&days=30
```

반환 형식 통일 예:
```json
{ "items": [{ "date": "2026-04-15", "title": "...", "url": "...", "kind": "주요사항보고서" }] }
```

## 5. 단계별 구현 계획

### Phase A — MVP (1차 스프린트)

1. `src/api/newsApi.js` 신규 — Yahoo `v1/finance/search?newsCount=8`로 뉴스 10건.
2. `src/api/technicalApi.js` 신규 — MA/RSI/MACD 계산 함수 3개.
3. `src/agents/researchAgent.js` 의 `buildResearchContext` 를 객체 번들형으로 확장.
4. `src/services/researchOrchestrator.js` 신규 — `Promise.allSettled` 병렬 조립.
5. `src/pages/Research.jsx` 또는 AI 채팅의 research 라우팅에서 오케스트레이터 결과를 `sendToAgent(..., { researchBundle }, 'research')`로 주입.
6. 티커 추출: 한글 → `searchKoreanStocks()`, 영문 → `fetchSearch()` 재활용.

### Phase B — 공시 확장

7. 무료 DART API 키 발급 + `.env`에 `DART_API_KEY` 보관.
8. `server/index.js`에 `/api/dart/list` 라우트 추가 (corp_code 매핑 캐시 포함).
9. `server/index.js`에 `/api/edgar/filings` 라우트 추가 (User-Agent 세팅).
10. `src/api/disclosureApi.js` 신규.
11. `buildResearchContext`에 `[최근 공시]` 섹션 추가.

### Phase C — Tool Use 리팩토링(선택)

12. 프록시를 `/api/claude`에서 `/api/claude/agentic`으로 확장하여 `tool_use → tool_result` 다중 라운드 지원.
13. 기존 오케스트레이터 함수들을 그대로 tool 핸들러로 재사용.
14. 모호한 질문(뉴스만, 실적만)에서 토큰 절감.

### Phase D — 품질 강화

15. 인용(`citations`) 구조화: 각 뉴스·공시 항목에 `source_url` 필수 포함.
16. 캐싱: React Query로 `researchBundle` 을 티커별 5~10분 캐시.
17. 레이트 리밋 대응: Yahoo/Naver 병렬 호출 시 5개씩 배치(이미 `fetchBatchQuotes` 패턴 있음).
18. PWA 오프라인: `vite.config.js` runtimeCaching에 `/api/dart`, `/api/edgar` StaleWhileRevalidate 규칙 추가.

## 6. 보안·규정 체크리스트

- [ ] DART API 키는 **서버 환경변수**로만 보관(`VITE_` 접두사 금지).
- [ ] SEC EDGAR `User-Agent` 헤더는 실제 연락 가능한 이메일 포함.
- [ ] Naver 크롤링을 채택할 경우 `robots.txt` 및 이용약관 재확인.
- [ ] Claude에 보내는 뉴스 제목에 **내부 PII 없음** 확인.
- [ ] 응답 말미에 "이 분석은 참고용이며 투자 결정의 책임은 본인에게 있습니다." 면책 문구 강제(이미 `RESEARCH_PROMPT`에 존재).
- [ ] 직접 매수/매도 지시성 문구는 프롬프트 레벨에서 금지.

## 8. 예상 토큰/지연시간

- 컨텍스트(5개 섹션 모두 포함): 대략 **1,500~2,500 input tokens**
- 응답(리포트): **1,500~2,500 output tokens** (현재 `AGENT_MAX_TOKENS.research = 3072` 충분)
- 병렬 페치 지연: 최대 소스(DART/EDGAR) ~800ms 예상 → 전체 1~2초
- Claude 응답: 3~8초 → **총 왕복 4~10초** (기존 research 호출과 유사)

- **이벤트 드리븐 알림**: 새 공시가 포착되면 AlertAgent에 자동 푸시.
- **포트폴리오 연계**: 보유 종목에 한해 매일 아침 종합 브리핑 생성 → Reports에 저장.
- **Multi-modal**: 차트 이미지(lightweight-charts 스크린샷)를 Claude Vision으로 해석.
- **사용자 프리퍼런스**: "나는 가치투자자" 같은 성향을 프롬프트에 주입해 해석 관점 커스터마이즈.

## 10. 요약

현재 앱은 **시세·재무는 있으나 뉴스·공시는 없음**, 그리고 **여러 소스를 한 번에 조립해 Claude에 주는 오케스트레이션 레이어가 없음**이 핵심 Gap.

권장 접근은 **A안 프리패치 → Phase A/B 순차 구현**이며, 기존 `/api/claude` 프록시와 `sendToAgent(..., 'research')` 호출 인터페이스를 유지한 채 `context.researchBundle` 만 추가하면 되므로 **기존 코드 파괴 없이 기능 확장**이 가능합니다. 이후 Phase C에서 Tool Use로 발전시키면 토큰 효율과 대화형 깊이 탐색까지 얻을 수 있습니다.
